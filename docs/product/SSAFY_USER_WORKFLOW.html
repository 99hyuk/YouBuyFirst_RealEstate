사용자 니즈

특정 지역의 흐름을 한눈에 보고, 실거래와 일정의 근거까지 보기 좋게 모아 확인하고 싶습니다.

가치 2 지도와 실거래로 사실 검증

지도는 지역 흐름을 한눈에 보여주고, 실거래 화면은 조건별 거래 목록과 공개 지연 상태를 함께 보여줍니다.

상세 화면은 기준일, 출처, 최신 여부, 근거 링크를 함께 보여줘 사용자가 정보의 한계를 확인할 수 있게 합니다.

워크플로우 1. 첫 접속 후 관심 지역 흐름 확인

상황 사용자는 서울·수도권 중 최근 흐름이 눈에 띄는 지역을 먼저 확인하고 싶습니다.
  1. 1대시보드에서 오늘의 3줄 브리핑을 읽습니다. 브리핑은 최근 시장 변화, 주목 지역, 관련 이슈를 짧은 문장으로 보여줍니다.
  2. 2오른쪽 주간 가격 흐름 영역에서 이번 주 많이 오른 지역과 많이 내린 지역을 확인합니다.
  3. 3검색창에 지역명이나 동 이름을 입력해 관심 지역을 찾고, 검색 결과를 눌러 지역 분석 지도로 이동합니다.
  4. 4지도에서 기간 탭을 바꾸며 상승, 하락, 자료 없음 상태를 비교합니다.
  5. 5지역 리포트 패널에서 기대 포인트, 우려 포인트, 기준일, 근거 출처를 확인합니다.
사용자 결과 오늘 먼저 살펴볼 지역을 고르고, 그 지역의 흐름이 어떤 근거를 바탕으로 표시됐는지 이해합니다.
화면에서 읽는 내용 지도와 리포트는 단순히 "좋다/나쁘다"를 말하지 않고, 사용자가 확인할 근거를 함께 보여줍니다.
  1. A지도 색상은 공개 반응이나 분위기가 아니라 공식 가격지수 또는 확인된 시장 데이터를 기준으로 합니다.
  2. B지역을 누르면 해당 지역의 요약 리포트가 열리고, 기대 요인과 우려 요인이 나뉘어 보입니다.
  3. C근거 출처 영역에서는 어떤 공식 자료나 데이터가 리포트에 쓰였는지 확인할 수 있습니다.
  4. D자료가 없거나 아직 공개되지 않은 지역은 임의 수치 대신 자료 없음, 공식 지수 미공표, 확인 필요로 표시합니다.
확인할 점 상승·하락 색상만 보는 서비스가 아니라 기준일과 근거를 같이 확인하게 만든다는 점을 강조합니다.

워크플로우 2. 실거래 화면에서 숫자로 검증

상황 사용자는 지도에서 본 지역 흐름이 최근 실거래와 전월세 데이터에서도 보이는지 확인하려고 합니다.
  1. 1상단 내비게이션의 실거래 또는 검색 결과의 법정동·단지 후보를 선택합니다.
  2. 2시도, 시군구, 매물 유형, 거래 방식, 비교 기간을 바꿔 조건에 맞는 거래만 좁혀 봅니다.
  3. 3지도 표시나 왼쪽 거래 목록을 눌러 선택한 거래의 요약과 가격 흐름을 봅니다.
  4. 4상세 패널에서 거래일, 면적, 가격, 주변 시설, 기준일, 공개 지연 상태를 확인합니다.
  5. 5로그인한 사용자는 하트 버튼으로 단지를 관심 목록에 저장하고, 말풍선 버튼으로 채팅에 첨부합니다.
사용자 결과 가격 흐름을 말로만 듣지 않고 실제 거래 조건과 거래 항목 기준으로 검증합니다.
예외 처리 공공데이터는 신고·공개 지연이 있을 수 있어 최신 거래처럼 단정하지 않습니다.
  1. A조건에 맞는 거래가 없으면 공식 거래 없음과 공개 지연 가능성을 표시합니다.
  2. B비교할 이전 데이터가 없으면 가격 변화율을 억지로 계산하지 않고 비교 데이터 없음으로 둡니다.
  3. C좌표가 불확실한 단지는 좌표 검증 전 상태로 표시하고 확정 위치처럼 말하지 않습니다.
  4. D상세 패널은 커뮤니티 반응보다 거래 요약, 가격 흐름, 데이터 기준일에 집중합니다.
확인할 점 필터를 바꿀 때 사용자의 질문이 "어디가 좋나"에서 "어떤 조건의 거래가 확인되나"로 바뀝니다.

워크플로우 3. 일정과 뉴스룸에서 근거 링크 확인

상황 사용자는 실거래·가격지수·청약·공급·정책 발표가 언제 있는지 확인하고 싶습니다.
  1. 1주요 일정 화면에서 월간 캘린더를 열고 가격·공시, 거래현황, 공급, 금융, 정책, 청약 일정을 봅니다.
  2. 2날짜 칸의 일정 줄을 눌러 하단 요약 행으로 이동합니다.
  3. 3공식 출처 링크가 있는 일정은 요약 행에서 실제 공표 상세, 청약 모집공고, 통화정책 일정으로 이동합니다.
  4. 4뉴스룸에서 뉴스, 리포트, 영상, 블로그·커뮤니티 링크를 종류별로 확인합니다.
  5. 5상세 리포트의 근거 출처와 뉴스룸 원문을 교차 확인합니다.

상황 사용자는 자주 보는 지역 또는 거래 상세를 저장해 다음 접속 때 빠르게 돌아오고 싶습니다.

  2. 2로그인에 성공하면 대시보드로 돌아오고 상단에 사용자 닉네임과 로그아웃 버튼이 표시됩니다.
  3. 3지역 분석 리포트나 실거래 상세에서 하트 버튼으로 관심 지역·단지를 저장합니다.

사용자 결과 관심 대상은 계정 기준으로, 최근 본 대상은 브라우저 기준으로 다시 이어볼 수 있습니다.

  3. C최근 본 항목은 같은 브라우저에서 빠르게 돌아가기 위한 임시 목록으로 사용합니다.
  4. D사용할 수 없는 간편 로그인 버튼은 비활성 상태로 보여줘 잘못된 화면으로 이동하지 않게 합니다.
확인할 점 로그인한 사용자는 관심 목록으로, 비로그인 사용자도 최근 본 목록으로 탐색을 이어갈 수 있음을 보여줍니다.

  3. 3지역 분석 리포트 또는 실거래 상세의 말풍선 버튼을 누르면 현재 대상이 채팅 입력창에 첨부됩니다.
  4. 4첨부 카드는 종류, 제목, 필요한 설명, 기간과 등락률을 압축해서 보여줍니다.
  5. 5전송된 첨부 카드를 클릭하면 원래 보던 지역 분석 또는 실거래 상세 화면으로 돌아갑니다.
사용자 결과 채팅이 단순 대화가 아니라 사용자가 본 리포트와 거래 상세를 다시 여는 탐색 보조 도구가 됩니다.

확인할 점 채팅은 단순한 대화창이 아니라 사용자가 보고 있던 지역과 거래를 함께 공유하는 도구입니다.

서비스 경계와 문구 원칙

행동 권유 금지

매수, 매도, 청약, 대출을 권유하지 않습니다. 상승 확정, 수익 보장, 진입 신호 같은 표현도 쓰지 않습니다.

공식 데이터 우선

실거래, 전세, 공급, 정책, 가격지수는 출처, 기준일, 최신 여부를 분리해 보여줍니다.

커뮤니티는 보조 관찰

공개 반응은 이슈 해석 보조 신호일 뿐 핵심 순위나 확정 판단의 기준으로 쓰지 않습니다.

원문 저장 제한

뉴스·리포트·공개 원문은 제목, 짧은 요약, 원문 링크, 출처, 시각 중심으로 제한 저장합니다.

불확실성 표시

확인할 수 없는 값은 확인 필요, 수집 전, 예시 데이터처럼 구분하고 임의로 채우지 않습니다.

근거 로그 노출

AI가 어떤 지표와 링크를 참고했는지 사용자용 요약과 주의 사항으로 남깁니다.

화면별로 사용자가 확인하는 정보

사용자 화면 주요 확인 정보 사용자가 얻는 답 주의해서 보는 부분

지역 분석 지도 지도 색상, 상승·하락 수치, 지역 리포트, 근거 출처 관심 지역이 주변 지역과 비교해 어떤 흐름인지 봅니다. 자료 없음, 공식 지수 미공표, 기준일을 함께 확인합니다.
실거래 거래 목록, 거래일, 면적, 가격, 비교 기간, 공개 지연 상태 지도에서 본 흐름을 실제 거래 기준으로 확인합니다. 거래가 없다는 말이 시장에 거래가 전혀 없다는 뜻은 아닐 수 있습니다.
주요 일정 통계 발표일, 청약 일정, 공급·정책·금리 발표, 공식 출처 언제 어떤 데이터를 다시 확인해야 하는지 알 수 있습니다. 날짜가 확인된 공식 일정만 실제 일정처럼 표시합니다.
