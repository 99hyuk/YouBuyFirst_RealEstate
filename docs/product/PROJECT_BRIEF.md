# 너나사 (YouBuyFirst) 프로젝트 기획 요약

## 제품 방향

너나사는 커뮤니티 반응, 시장 시세, AI 분석, 모의투자를 결합해 "대중보다 먼저 사고, 대중과 반대로 검증한다"는 아이디어를 실험하는 투자 참고형 시뮬레이터입니다. 완제품 기준은 `docs/product/FINAL_PRODUCT_PLAN.md`입니다.

현재 구현은 완제품의 첫 기반인 커뮤니티 수집/저장 파이프라인에서 시작했지만, 앞으로의 업무는 단순 MVP가 아니라 완제품 사용 루프를 세로 slice로 완성하는 방식으로 뽑습니다.

## 현재 구현 기반

- Spring Boot가 중심 시스템이며 저장, 검증, 중복 제거, admin/public API를 담당합니다.
- Python pipeline은 크롤링, Playwright fallback, LLM 분석, provider adapter를 담당합니다.
- MySQL은 커뮤니티 글, 종목 metric, crawl run, market snapshot, 이후 원장 데이터를 저장합니다.
- Vue 3 + Vite + TypeScript front는 mock 화면과 API 후보를 먼저 세우고, backend/pipeline 계약을 역으로 도출합니다.
- `OPENAI_API_KEY`가 없으면 mock provider로 로컬 시연을 가능하게 합니다.

## 완제품 사용 루프

1. 관심종목에서 가격, 거래량, 커뮤니티 반응, 기사/공시 변화가 감지됩니다.
2. 사용자는 변화 원인 후보, 근거 링크, 신뢰도/주의 배지를 확인합니다.
3. 종목 상세에서 시세, 커뮤니티 반응, 뉴스/공시/인기글, 이벤트 타임라인을 봅니다.
4. 인간 지표 화면에서 개미 심리 지수, 통합 커뮤니티 반응, paper 성과 비교를 확인합니다.
5. 모의 포트폴리오와 AI 에이전트 판단 로그로 판단 이후의 가격/반응 흐름을 복기합니다.

## 우선 완성할 세로 Slice

- 관심종목 브리핑: 종목별 가격, 거래량, 언급량, 개미 심리 지수, 새 이벤트를 한 화면에 압축합니다.
- 종목 상세: quote/chart/수급, 팩트폭격 헤드라인, 커뮤니티 반응, 근거 링크, 이벤트 타임라인을 연결합니다.
- 인간 지표: 커뮤니티 반응 ranking, 개미 심리 지수, 통합 지표 기반 성과 실험, 에이전트 판단 로그를 묶습니다.
- 모의 포트폴리오: 가상 예수금, 주문, 체결, 원장, 포지션, 손익, 체결 후 복기를 트랜잭션 정합성 중심으로 만듭니다.
- AI 에이전트: 같은 통계 window와 strategy version 기준으로 판단하고, 중복 판단/중복 주문을 막습니다.

## 데이터 전략

- 커뮤니티는 일반 게시판형과 종목 게시판형을 분리합니다.
- 에펨코리아 같은 일반 게시판형 소스는 최근 글 목록을 수집한 뒤 글 안에서 종목을 인식합니다.
- 에펨코리아는 로컬 개인 지표용 source로만 다루며, 수집이 허용된 로컬 환경에서도 HTTP 목록 요청을 먼저 보내지 않고 브라우저 렌더링으로 접근합니다.
- 네이버 종토방 같은 종목 게시판형 소스는 모든 종목을 전수 순회하지 않고, 우선순위가 있는 `CrawlTarget` 큐로 관리합니다.
- 뽐뿌 증권포럼, 디시 미국주식/주식갤러리/국내주식 계열은 source registry 후보로 두되, 공개 배포 가능 범위를 소스 상태로 분리합니다.
- 시장 데이터는 `yfinance` + FinanceDataReader 조합을 기본 후보로 둡니다. 공개 화면에는 provider, `asOf`, 지연, stale, dataStatus를 함께 표시합니다.
- 뉴스/공시/영상/블로그/인기글은 원문 대체 화면이 아니라 외부 링크와 이벤트 원인 후보를 보여주는 보조 데이터입니다.

## 정책 경계

- 실제 투자 자문, 실거래 지시, 수익 보장, 개인화 투자 권유처럼 보이는 기능과 표현은 피합니다.
- 로그인, CAPTCHA, 프록시, fingerprint 우회는 하지 않습니다.
- 공개 화면은 원문보다 집계 지표, 대표 키워드, 반응 방향 비율, AI 재서술 근거를 중심으로 설계합니다.
- 공개 화면은 커뮤니티 이름별 성향 비교나 source별 수익률 우열을 직접 보여주지 않고, 통합 지표와 반응 일관성/신뢰도/편중 주의로 압축합니다.
- 제한 원문은 제목, 본문 일부, URL, 작성 시각, 작성자 표시명 hash, 원문 hash 중심으로 저장하고 보관 기간을 제한합니다.

## 후순위 확장

- OCR 자산 연동과 S3 Presigned URL 업로드
- 텔레그램 또는 앱 알림
- Spring Security 인증/인가
- Redis quote cache와 WebSocket 실시간 브로드캐스트
- 사용자 반응방/게시판/채팅
- 벡터DB 기반 유사 상황 검색과 질문형 분석
- 부동산 별도 버티컬
