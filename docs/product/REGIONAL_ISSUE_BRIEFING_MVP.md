# 부동산 인사이트 탐색 MVP

작성일: 2026-06-22

## 결론

시연 MVP는 `개별 아파트 전체 커버리지`나 `커뮤니티 반응 크롤링`이 아니라 `지도 기반 지역 흐름`, `실거래 탐색`, `주요 일정`, `근거 브리핑`을 중심으로 완성한다.

너나사 부동산은 사용자가 접속했을 때 "어느 지역의 실거래·전세·공급·정책 흐름을 지금 확인해야 하는지"를 빠르게 이해하게 하는 서비스다. 따라서 1차 완성 기준은 특정 단지 하나하나를 모두 크롤링하고 해석하는 것이 아니라, 시군구, 읍면동, 생활권 단위로 공식 데이터와 이슈 근거를 묶어 AI가 짧은 근거 리포트를 만들어 주는 흐름이다.

## 왜 바꾸는가

- 부동산은 단지가 너무 많고 커뮤니티 언급도 흩어져 있어, 단지 단위 전체 커버리지를 단기간에 안정적으로 만들기 어렵다.
- 발표와 시연에서는 "실제로 서비스처럼 보이는 화면"이 더 중요하다. 빈 단지 리포트가 많으면 완성도가 낮아 보인다.
- 부동산 시장은 실제로도 동, 구, 생활권, 교통권, 정책 영향권 단위로 같이 움직이는 경우가 많다.
- 공공데이터, 실거래/전월세, 가격지수, 공급 일정, 정책 이벤트는 사용자가 실제로 확인할 이유가 강하다.
- SerpApi, 뉴스, 리포트, 영상, 블로그는 지역 이슈 후보와 근거 링크를 빠르게 채우기에 적합하다.
- 커뮤니티 크롤링은 핵심 정체성에서 내리고, 1차 MVP에서는 보조 쟁점 후보로만 쓴다.

## MVP 해석 단위

| 우선순위 | 단위 | 화면 의미 |
| --- | --- | --- |
| P0 | 시군구, 읍면동, 생활권 | "지금 어느 지역의 시장 사실을 봐야 하는지"를 보여주는 기본 리포트 단위 |
| P1 | 시도 | 지도 roll-up, 필터, 큰 흐름 요약 |
| P2 | 단지/아파트 | 좌표와 공식 실거래가 확인된 경우에만 보조 증거로 노출 |

단지/아파트는 버리지 않는다. 다만 "모든 단지 분석"이나 "단지 반응 랭킹"을 MVP 완료 기준으로 두지 않는다. 단지 데이터는 지역 리포트 안의 실거래 근거, 후보, 상세 링크로 점진 확장한다.

## 데이터 우선순위

1. 공공데이터와 공식 지표
   - 지역별 가격지수, 실거래/전월세, 미분양, 공급, 인허가, 정책 이벤트
   - provider, asOf, stale, 데이터 없음 상태를 분리한다.
2. 최근 이슈 후보
   - SerpApi, 뉴스, 리포트, 영상, 블로그를 통해 정책, 교통, 공급, 전세, 금리, 개발 이슈 후보를 수집한다.
   - 검색 결과 수 자체를 관심도 점수로 쓰지 않고, 근거 링크 후보로만 쓴다.
3. 실거래 탐색
   - 공식 매매/전월세 거래를 지역, 기간, 거래유형, 가격대, 면적 조건으로 필터링한다.
   - 거래일, 기준월, 공개/신고 지연, provider, 적재 시각을 분리한다.
   - 공식 데이터가 없으면 빈 상태와 공개 지연 가능성을 표시한다.
4. AI 지역 브리핑
   - 위 데이터를 묶어 지역별로 "무슨 시장 사실이 있고, 어떤 근거가 있으며, 데이터 한계는 무엇인지"를 요약한다.
5. 보조 반응 신호
   - 공개 접근 가능한 source만 후보로 둔다.
   - 지역 언급, 생활권 표현, 대표 단지 언급은 핵심 순위가 아니라 보조 쟁점 후보로만 쓴다.

## 화면 기준

### 대시보드

- 첫 화면은 브랜드, 검색, 오늘의 지역 인사이트, 지도/실거래 탐색 진입점, 주요 일정, 최신 콘텐츠 feed를 보여준다.
- 단지 후보는 있으면 보조로 보여주되, 빈 단지 분석 때문에 화면이 비어 보이게 만들지 않는다.

### 지도

- 전국, 시도, 시군구 heatmap은 지역별 가격 흐름과 이슈 강도를 중심으로 표시한다.
- 지역별 공식 지표가 없는 곳은 검정에 가까운 회색으로 `데이터 없음/공식 지수 미공표`처럼 구분한다.
- 지도에서 지역을 누르면 해당 지역 리포트로 이어진다.

### 실거래 탐색

- 기존 지역 반응 화면은 실거래 탐색 화면으로 전환한다.
- 지역, 기간, 거래유형, 가격대, 면적 필터를 기본으로 둔다.
- 단지 TOP10은 표시하지 않는다. 단지는 거래 row나 상세 링크 안에서만 보조로 다룬다.

### 뉴스룸

- 뉴스, 정책·통계 리포트, 영상, 블로그/커뮤니티 링크를 최신순으로 채운다.
- 지역이 특정되지 않은 전국 이슈도 보여준다.
- 리포트 feed는 언론 기사와 분리해 금융사, 공공기관, 연구기관 자료를 우선한다.

### 지역/단지 상세

- 기본은 지역 상세 리포트다.
- 단지 상세는 좌표, 실거래, 공식 provider key, 충분한 언급이 있을 때만 풍부하게 보여준다.
- 데이터가 부족하면 억지 분석 대신 `수집 전`, `표본 부족`, `좌표 검증 전`으로 남긴다.

## 구현 우선순위

1. 화면 문구와 빈 섹션 정리
   - 크롤링/지역 반응 중심 문구를 줄이고, 실거래 탐색과 지역 인사이트 중심으로 맞춘다.
2. 뉴스룸과 대시보드 콘텐츠 feed 보강
   - 뉴스, 리포트, 영상, 블로그/커뮤니티를 유형별로 채운다.
3. 지도와 실거래 탐색 연결
   - 지역별 상승/하락, 실거래 row, 이슈 후보, AI 브리핑을 같은 targetId로 연결한다.
4. 일일 AI 지역 브리핑
   - 공공데이터, 최근 이슈 후보, 커뮤니티 보조 신호를 입력으로 EvidenceLog를 생성한다.
5. 주요 일정 화면
   - 가격지수, 실거래, 미분양·공급, 금리, 청약, 정책 발표 일정을 캘린더로 정리한다.
6. 단지/아파트 상세 보강
   - 공식 실거래, 좌표, alias가 연결된 단지만 상세 품질을 올린다.

## 하지 않는 것

- 모든 아파트 단지를 1차 MVP에서 분석 완료하지 않는다.
- 커뮤니티 반응을 핵심 순위나 서비스 정체성으로 세우지 않는다.
- 로그인 세션 크롤링, CAPTCHA 우회, 프록시 회전, fingerprint 위장은 하지 않는다.
- 검색 노출량을 관심도 점수로 쓰지 않는다.
- 가격 상승을 단정하거나 매수, 매도, 청약, 대출 행동을 권유하지 않는다.

## 완료 기준

- 대시보드가 브랜드, 지역 인사이트, 지도/실거래 탐색, 주요 일정, 뉴스룸 feed까지 비어 보이지 않는다.
- 지도에서 지역을 눌렀을 때 해당 지역의 가격/이슈/AI 브리핑이 연결된다.
- 실거래 탐색은 지역, 기간, 거래유형, 가격, 면적 조건으로 공식 거래를 조회할 수 있다.
- 뉴스룸은 뉴스, 정책·통계 리포트, 영상, 블로그/커뮤니티가 유형별로 구분된다.
- 지역 상세는 EvidenceLog, 최근 이슈, 시장 사실, 데이터 한계를 함께 보여준다.
- 단지 상세는 가능한 경우에만 좌표와 최근 거래를 보여주고, 부족하면 수집 전 상태를 숨기지 않는다.
