# 너나사 부동산 최종 기획안

## 제품 한 줄 설명

너나사 부동산은 실거래, 전세, 매물, 공급, 정책 이슈를 지역 기준으로 엮어 보여주는 AI 부동산 인사이트 탐색 서비스입니다.

단순히 실거래가 표나 뉴스 목록만 보여주는 서비스가 아니라 공식 거래 데이터, 전세 흐름, 공급 일정, 정책·교통 이슈, 리포트 근거를 같은 지역 target에 묶어 사용자가 직접 탐색하고 해석하게 합니다. 단지/아파트는 공식 좌표와 실거래가 확인된 경우 지역 리포트의 보조 근거와 상세 화면으로 확장합니다. 공개 커뮤니티 반응은 핵심 엔진이 아니라 이슈 해석을 보조하는 관찰 신호로만 둡니다.

핵심 구현 범위는 `docs/product/REGIONAL_ISSUE_BRIEFING_MVP.md`, `docs/product/real-estate-one-page-plan.html`, `docs/product/CORE_IMPLEMENTATION_SCOPE.md`를 기준으로 합니다. 한페이지 기획안의 핵심은 `지역 흐름 + 실거래 탐색 + 시장 사실 + 주요 일정 + 에이전트 근거`이며, 크롤링 서비스가 아니라 공식 데이터 기반 인사이트 도구입니다.

## 제품 목표

실거래가 표와 매물 목록만 보여주는 도구가 아니라, 어떤 지역의 가격·전세·공급·정책 흐름을 지금 확인해야 하는지 보여줍니다.

부동산은 지역/생활권/정책 영향권/검증 단지 후보로 대상이 잘게 나뉘며 지역 단위 상승과 정책 이벤트에 민감합니다. 시간 제한이 있는 MVP에서는 지도 기반 지역 흐름과 실거래 탐색을 먼저 완성하고, 단지/아파트는 검증 가능한 데이터가 있는 대상부터 보조 리포트로 확장합니다. 그래서 target hierarchy, market fact, policy timeline, market data schedule, source registry를 1차 구현 범위로 둡니다.

이 제품은 특정 매수, 매도, 청약, 대출 행동을 권유하지 않습니다. 운영 단계에서도 가격 상승 단정, 수익 보장, 개인화 행동 권유처럼 보이는 기능과 표현은 배제합니다.

## 핵심 사용자

- 동, 구, 생활권별 가격·전세·공급 흐름을 빠르게 보고 싶은 사용자
- 실거래와 전월세 데이터를 조건별로 탐색하고 싶은 사용자
- 정책, 교통, 공급, 전세, 재건축 이슈가 어느 지역에 연결되는지 보고 싶은 사용자
- 공공데이터와 부동산 시장 사실을 연결하는 데이터 파이프라인을 기술 경험으로 설명하고 싶은 개발자

## 일일 사용 루프

1. 사용자는 대시보드에서 오늘 확인할 지역 흐름과 핵심 뉴스/리포트를 확인합니다.
2. 지도에서 전국과 시군구 흐름을 공간적으로 확인합니다.
3. 실거래 탐색에서 지역, 기간, 거래유형, 가격대, 면적 조건으로 공식 거래 데이터를 좁혀 봅니다.
4. 주요 일정에서 가격지수, 실거래 공개, 미분양·공급, 금리, 청약, 정책 발표 시점을 확인합니다.
5. 지역 상세에서 실거래/전세/매물 흐름과 정책·뉴스 근거를 하나의 타임라인에서 봅니다.
6. 에이전트 평가와 근거 로그로 어떤 데이터가 판단에 쓰였는지 확인합니다.

## 핵심 판단 구조

```text
공식 실거래/전세/매물/정책 데이터 수집
-> market fact 정규화
-> 지역/생활권/검증 단지 target 연결
-> 지도 layer와 실거래 탐색 API 구성
-> SerpApi/뉴스/리포트/공식 일정으로 최근 이슈 후보 탐색
-> 에이전트가 시장 사실 + 주요 일정 + 최근 이슈 + 보조 반응 신호를 함께 보고 평가
-> 지역/단지 평가, 근거 링크, 판단 로그 저장
```

핵심 해석 단위는 `지역 target + 기간 + market fact`입니다. MVP에서는 시군구, 읍면동, 생활권을 우선하고 단지는 검증된 경우에만 보조 target으로 붙입니다. 실거래와 전월세는 거래일, 신고/공개 지연, 기준월, 적재 시각을 분리하고, 쟁점은 교통, 학군, 전세, 재건축, 청약, 대출, 공급, 정책, 개발 호재, 가격 부담, 실거주 만족도처럼 사용자가 이유를 이해할 수 있는 라벨로 묶습니다.

벡터DB는 지역/단지를 식별하거나 실거래가를 계산하는 용도가 아닙니다. 과거 시장 흐름, 정책 이벤트, 리포트 근거를 검색해 현재 지역 브리핑의 맥락을 보강하는 용도입니다.

SerpApi 같은 검색 API는 관심도 점수의 원천이 아닙니다. 검색 노출은 편향이 크므로 최근 뉴스, 정책, 개발, 교통, 금리, 대출, 청약 이슈 후보와 근거 링크를 보강하는 보조 채널로만 사용합니다. 저장 범위는 제목, 출처, 날짜, 링크, 관련 키워드 중심으로 제한합니다.

## 제품 용어 기준

- 사용자 화면의 대표 표현은 `실거래`, `전세`, `공급`, `정책`, `시장 사실`, `공개 지연`, `인사이트`, `근거 로그`입니다.
- 공개 원문에서 특정 지역/단지를 바라보는 방향은 보조 `반응 방향`이라고 부릅니다.
- 여러 글을 지역/단지/window 단위로 묶은 결과는 핵심 지표가 아니라 보조 `반응 관찰`로 부릅니다.
- 화면 결론은 `거래 확인`, `공식 데이터 대기`, `전세 압력`, `공급 일정`, `정책 이벤트`처럼 사용자가 직접 확인할 수 있는 상태로 표현합니다.
- `추천`, `사라`, `팔아라`, `수익 보장`, `진입`, `시그널 확정`, `오를 지역`, `청약 넣어라`, `대출 받아라` 같은 행동 지시 표현은 쓰지 않습니다.

## 최종 핵심 기능

### 0. 핵심 구현 축

| 축 | 기술 흐름 | 산출물 |
| --- | --- | --- |
| 지도 흐름 | 가격지수/market fact -> 지역 target 연결 -> map layer snapshot | 전국/시군구 흐름 지도 |
| 실거래 탐색 | 공공데이터/API -> 거래 row 정규화 -> 필터/정렬 API | 매매·전월세 탐색 테이블 |
| 시장 사실 | 공공데이터/API -> 시점별 저장 -> 이벤트 타임라인 -> 변화 감지 | 가격/전세/매물 흐름, 뉴스/정책 연결 |
| 주요 일정 | 공식 출처/반복 일정 -> 일정 캘린더 -> source link | 가격지수, 실거래, 공급, 금리, 청약 일정 |
| 보조 반응 | 공개 source 후보 -> 제한 저장 -> 쟁점 후보 보강 | 보조 관찰 키워드 |
| 에이전트 판단 | 지표 입력 -> AI 근거 검색 -> 맥락 종합 -> 근거 로그 | 지역/단지 평가와 caveat |

### 1. 부동산 인사이트 대시보드

대시보드는 첫 화면부터 실제 제품 화면이어야 합니다. 소개 페이지가 아니라 오늘 확인해야 할 지역 흐름, 공식 데이터 상태, 최근 이슈, 다음 확인 일정을 바로 보여줍니다.

주요 섹션:

- 오늘의 지역 인사이트와 공식 데이터 상태
- 지도 흐름, 실거래 탐색, 주요 일정으로 이어지는 진입점
- 쟁점 라벨: 교통, 학군, 전세, 재건축, 청약, 대출, 공급, 정책
- 표본 수, 소스 편중, 수집 지연, provider/asOf/stale 배지
- 최신 뉴스/리포트/공식 출처 링크

### 2. 실거래 탐색

실거래 탐색은 사용자가 지역, 기간, 거래유형, 가격대, 면적 조건으로 공식 거래 데이터를 좁혀 보는 화면입니다. 특정 단지를 추천하지 않고 거래 row와 지역 집계, 공개 지연 상태를 보여줍니다.

주요 섹션:

- 필터: 지역, 기간, 매매/전월세, 가격대, 면적, 데이터 출처
- 요약: 최근 거래 수, 중위 거래가, 전세 거래 수, 데이터 기준일
- 결과 테이블: 지역/단지명, 거래일, 면적, 가격, 층, 거래유형, provider
- 상태: 공식 거래 없음, 공개 지연 가능, 데이터 수집 전

### 3. 지역 상세 타임라인

지역을 클릭하면 시장 사실과 근거 링크를 같은 시간축에서 보여줍니다. 단지 상세는 좌표, 실거래, 공식 provider key가 연결된 경우에만 풍부하게 보여줍니다.

주요 섹션:

- 지역 기본 정보와 alias
- 실거래/전세/매물 흐름
- 정책/개발/교통 이벤트
- 뉴스/컬럼 링크
- 보조 반응 키워드
- 데이터 한계와 근거 로그

### 4. 주요 일정

주요 일정은 가격지수, 실거래 공개, 미분양·공급, 금리, 청약, 정책 발표를 캘린더로 보여주는 화면입니다. 사용자는 언제 어떤 공식 데이터를 다시 확인해야 하는지 판단합니다.

필수 표시 기준:

- provider
- scheduleDate
- sourceUrl
- lastCheckedAt
- `stale`
- 지연 여부
- 값이 없을 때 `unknown`, `null`, `확인 필요`, `mock` 구분

### 5. 유사 과거 상황 비교

비슷한 시장 사실과 정책 이벤트가 있었던 과거 window를 찾아 이후 시장 흐름을 비교합니다. 이 기능은 미래를 예측하거나 행동을 지시하는 기능이 아니라, 과거 관찰 사례를 함께 보여주는 분석 레이어입니다.

후보 입력:

- market fact timeline
- issueMix
- 대표 뉴스/정책 이벤트
- 이후 실거래/전세/매물 흐름

### 6. 에이전트 평가와 근거 로그

에이전트는 지역/단지 상태를 평가하되 행동 지시를 하지 않습니다. 사용자에게는 짧은 평가, 근거 요약, caveat, 데이터 상태만 보여줍니다.

평가 입력:

- 시장 사실: 실거래가, 전세가, 매물량, 청약, 정책 데이터의 기준 시각
- 데이터 상태: 공식 데이터 있음, 공식 데이터 없음, 공개 지연, 부분 반영
- 주요 쟁점: 교통, 학군, 전세, 재건축, 청약, 대출, 공급, 정책 등
- 유사 과거: 비슷한 market fact와 이후 실거래/전세/매물 흐름
- 최근 이슈: 뉴스, 컬럼, 정책, 개발, 금리, 대출, 교통 이슈 후보
- 보조 반응: 공개 원문에서 반복되는 쟁점 후보

평가 출력:

- 대상: 지역 또는 단지
- 시장 사실 요약
- 데이터 한계와 주요 쟁점
- 유사 과거 비교
- 최근 이슈 후보
- 시장 사실 요약과 `provider/asOf/stale`
- 종합 평가
- 근거 로그: 사용한 지표, 링크, snapshot id, 검색 provider, model version, prompt version

기록 후보:

- `targetType`, `targetId`
- `evaluatedAt`
- `summary`
- `evidence`
- `caveats`
- `source/asOf/stale`
- `strategyVersion` 또는 `evaluationVersion`
- `skipReason`

## 주요 데이터 도메인

- RealEstateRegion: 시/구/동/생활권 같은 지역
- RealEstateComplex: 아파트/주거 단지
- RealEstateTargetGraph: 지역, 생활권, 정책 영향권, 단지의 상하위/영향 관계
- RealEstateAlias: 공식명, 줄임말, 생활권 표현, 공개 원문 보조 별칭
- CrawlSource: 공공데이터, 뉴스/컬럼, 공개 게시판 보조 source registry
- CommunityPost: 보조 관찰용으로 제한 저장된 게시글 메타데이터와 snippet
- RealEstateMention: 공개 원문 안에서 인식된 지역/단지 언급
- ReactionAnalysis: 보조 언급별 반응 방향과 쟁점
- RealEstateReactionSnapshot: 지역/단지/window 단위 보조 반응 관찰
- RealEstateMarketFact: 실거래/전세/매물/정책/공급/뉴스 상태
- RealEstatePolicyEvent: 정책, 교통, 공급, 대출, 청약, 학군, 재건축 이벤트와 영향 범위
- RealEstateTimelineEvent: 반응, 뉴스, 정책, market fact를 시간순으로 묶은 이벤트
- EvidenceLog: 에이전트 평가와 근거 기록

- 유사 과거 시장 흐름 검색 후보

### Phase 5. 에이전트 근거 로그

- 지역/단지 평가 입력 contract
- evidence log 저장 모델
- 사용자용 평가 문구 기준
- 내부 추론 비노출 기준

## 명시적 제외 및 주의 사항

- 특정 매수, 매도, 청약, 대출 행동을 권유하지 않습니다.
- 가격 상승, 수익, 청약 성공을 단정하지 않습니다.
- 커뮤니티 로그인, CAPTCHA 회피, 프록시 우회, fingerprint 위장은 하지 않습니다.
- 원문 대량 저장과 외부 원문 재배포는 하지 않습니다.
- 작성자 개인별 성향 추적, 닉네임 랭킹, 개인 프로필 수집은 하지 않습니다.
- 내부 chain-of-thought는 저장하거나 노출하지 않고, 사용자용 짧은 근거만 저장합니다.

## 작업 영역과 정본 위치

- `realestate` 작업: 지역/단지, market fact, 정책 이벤트, alias 문서를 기준으로 합니다.
- `community` 작업: 보조 공개 수집, source policy, 원문 제한 저장 문서를 기준으로 합니다.
- `indicator` 작업: 보조 반응 방향, 쟁점 비율, snapshot, 표본 신뢰도 문서를 기준으로 합니다.
- `agent` 작업: 지역/단지 평가, 근거 로그, 유사 과거 비교 설명을 기준으로 합니다.
- `ui` 작업: `layers/ui` 문서와 화면별 Screen Brief를 기준으로 합니다.
- `ops` 작업: `layers/ops` 문서를 기준으로 합니다.
- 기존 금융 서비스 전용 영역은 active runtime에서 제거했고, 필요한 맥락은 git history와 결정 로그로만 확인합니다.
