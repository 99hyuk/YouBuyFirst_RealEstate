# 너나사 부동산 최종 기획안

## 제품 한 줄 설명

너나사 부동산은 수요자 반응 기반 부동산 시장 해석 서비스입니다.

단순히 실거래가, 매물, 뉴스만 보여주는 서비스가 아니라 실제 사람들이 커뮤니티, 댓글, 뉴스/컬럼 반응에서 남기는 관심, 기대, 불안, 의심을 지역/단지 기준 지표로 만들고 시장 사실 데이터와 함께 해석합니다.

핵심 구현 범위는 `docs/product/real-estate-one-page-plan.html`과 `docs/product/CORE_IMPLEMENTATION_SCOPE.md`를 기준으로 합니다. 한페이지 기획안의 핵심은 `반응 지표 + 유사 과거 + 에이전트 판단`이며, 가격표가 아니라 시장 심리와 시장 사실을 함께 추적하는 것입니다.

## 제품 목표

실거래가 표와 매물 목록만 보여주는 도구가 아니라, 사람들이 어떤 지역과 단지를 왜 말하는지, 그 반응이 정책/교통/전세/공급/뉴스 흐름과 어떻게 연결되는지 보여줍니다.

부동산은 지역/단지/생활권/정책 영향권으로 대상이 잘게 나뉘며 지역 단위 상승과 정책 이벤트에 민감합니다. 그래서 target hierarchy, alias DB, policy timeline, source registry를 1차 구현 범위로 둡니다.

이 제품은 특정 매수, 매도, 청약, 대출 행동을 권유하지 않습니다. 운영 단계에서도 가격 상승 단정, 수익 보장, 개인화 행동 권유처럼 보이는 기능과 표현은 배제합니다.

## 핵심 사용자

- 지역과 단지별 관심 변화와 이슈를 빠르게 보고 싶은 사용자
- 커뮤니티 반응, 뉴스/컬럼, 시장 사실을 한 화면에서 비교하고 싶은 사용자
- 정책, 교통, 공급, 전세, 재건축 이슈가 어느 지역에 집중되는지 보고 싶은 사용자
- 커뮤니티 데이터와 부동산 시장 사실을 연결하는 데이터 파이프라인을 기술 경험으로 설명하고 싶은 개발자

## 일일 사용 루프

1. 사용자는 대시보드에서 요즘 언급이 많은 지역과 단지를 확인합니다.
2. 지역/단지 상세에서 언급량, 기대/우려, 쟁점 비율, 표본 신뢰도를 봅니다.
3. 뉴스/컬럼 링크, 정책/개발/교통 이벤트, 실거래/전세/매물 흐름을 하나의 타임라인에서 봅니다.
4. 비슷한 과거 반응 상황과 이후 시장 흐름을 비교합니다.
5. 에이전트 평가와 근거 로그로 어떤 데이터가 판단에 쓰였는지 확인합니다.

## 핵심 판단 구조

```text
커뮤니티/댓글/뉴스/컬럼 수집
-> 지역/단지 언급 식별
-> 언급량, 반응 방향, 쟁점 비율, 표본 신뢰도 지표화
-> 벡터DB 또는 batch similarity로 과거 유사 반응 패턴과 이후 시장 흐름 검색
-> SerpApi 같은 검색 API로 최근 뉴스, 정책, 개발, 교통, 금리 이슈 후보 탐색
-> 에이전트가 반응 지표 + 유사 과거 + 최근 이슈 + 시장 사실을 함께 보고 평가
-> 지역/단지 평가, 근거 링크, 판단 로그 저장
```

핵심 해석 단위는 `지역/단지 target + window`입니다. 언급량 지표는 최근 증가율, 반복 언급, source 다양성, 표본 수를 함께 보고, 반응 방향은 기대/우려/중립/혼조로 분류합니다. 쟁점은 교통, 학군, 전세, 재건축, 청약, 대출, 공급, 정책, 개발 호재, 가격 부담, 실거주 만족도처럼 사용자가 이유를 이해할 수 있는 라벨로 묶습니다.

벡터DB는 지역/단지를 식별하거나 실거래가를 계산하는 용도가 아닙니다. 현재 반응 패턴과 과거 반응 window를 비교해 이후 실거래가, 전세가, 매물 흐름이 어떻게 달라졌는지 설명하는 과거 유사 상황 검색용입니다.

SerpApi 같은 검색 API는 관심도 점수의 원천이 아닙니다. 검색 노출은 편향이 크므로 최근 뉴스, 정책, 개발, 교통, 금리, 대출, 청약 이슈 후보와 근거 링크를 보강하는 보조 채널로만 사용합니다. 저장 범위는 제목, 출처, 날짜, 링크, 관련 키워드 중심으로 제한합니다.

## 제품 용어 기준

- 사용자 화면의 대표 표현은 `관찰`, `분석`, `반응 지표`, `쟁점 비율`, `표본 신뢰도`, `근거 로그`입니다.
- 단일 글에서 특정 지역/단지를 바라보는 방향은 `반응 방향`이라고 부릅니다.
- 여러 글을 지역/단지/window 단위로 묶은 결과는 `지역 반응 지표` 또는 `단지 반응 지표`라고 부릅니다.
- 화면 결론은 `기대 우세`, `우려 우세`, `혼조`, `관망`처럼 관찰 상태로 표현합니다.
- `추천`, `사라`, `팔아라`, `수익 보장`, `진입`, `시그널 확정`, `오를 지역`, `청약 넣어라`, `대출 받아라` 같은 행동 지시 표현은 쓰지 않습니다.

## 최종 핵심 기능

### 0. 핵심 구현 축

| 축 | 기술 흐름 | 산출물 |
| --- | --- | --- |
| 커뮤니티 반응 | 크롤링 수집 -> DB 저장 -> 임베딩/쟁점 묶기 -> 지표화 | 관심 지수, 쟁점 비율, 표본 신뢰도 |
| 시장 사실 | 공공데이터/API -> 시점별 저장 -> 이벤트 타임라인 -> 변화 감지 | 가격/전세/매물 흐름, 뉴스/정책 연결 |
| 비슷한 과거 | 과거 사례 저장 -> 유사 사례 DB -> 유사 패턴 검색 -> 결과 비교 | 과거 비교 리포트 |
| 에이전트 판단 | 지표 입력 -> AI 근거 검색 -> 맥락 종합 -> 근거 로그 | 지역/단지 평가와 caveat |

### 1. 부동산 반응 대시보드

대시보드는 첫 화면부터 실제 제품 화면이어야 합니다. 소개 페이지가 아니라 요즘 언급 많은 지역/단지, 실시간 이슈, 쟁점 비율, 데이터 상태를 바로 보여줍니다.

주요 섹션:

- 요즘 언급 많은 지역/단지
- 기대/우려 방향과 mention count 변화
- 쟁점 비율: 교통, 학군, 전세, 재건축, 청약, 대출, 공급, 정책
- 표본 수, 소스 편중, 수집 지연, provider/asOf/stale 배지
- 최신 뉴스/컬럼/커뮤니티 링크

### 2. 지역/단지 상세 타임라인

지역 또는 단지를 클릭하면 반응 지표와 시장 사실을 같은 시간축에서 보여줍니다.

주요 섹션:

- 지역/단지 기본 정보와 alias
- 최근 reaction snapshot
- 주요 쟁점과 대표 근거 snippet
- 실거래/전세/매물 흐름
- 정책/개발/교통 이벤트
- 뉴스/컬럼 링크
- 유사 과거 상황과 이후 시장 흐름

### 3. 커뮤니티 반응과 쟁점 분석

커뮤니티 글과 댓글, 뉴스/컬럼 반응을 지역/단지 기준으로 묶습니다. 단순 긍정/부정 감정이 아니라 기대, 우려, 중립, 쟁점 비율, 표본 신뢰도를 함께 봅니다.

지표 후보:

- 언급량 변화
- 기대/우려/중립 비율
- 쟁점 비율
- 확산 강도
- source 다양성
- 표본 수 신뢰도
- source skew
- 수집 지연 여부

### 4. 시장 사실 데이터

실거래/전세/매물/정책 데이터는 반응 지표의 보조 근거이자 별도 관찰 데이터입니다.

필수 표시 기준:

- provider
- `asOf`
- `stale`
- 지연 여부
- 원천 링크 또는 출처
- 값이 없을 때 `unknown`, `null`, `확인 필요`, `mock` 구분

### 5. 유사 과거 상황 비교

비슷한 반응과 쟁점 비율이 있었던 과거 window를 찾아 이후 시장 흐름을 비교합니다. 이 기능은 미래를 예측하거나 행동을 지시하는 기능이 아니라, 과거 관찰 사례를 함께 보여주는 분석 레이어입니다.

후보 입력:

- reaction snapshot
- issueMix
- 대표 뉴스/정책 이벤트
- market fact timeline
- 이후 실거래/전세/매물 흐름

### 6. 에이전트 평가와 근거 로그

에이전트는 지역/단지 상태를 평가하되 행동 지시를 하지 않습니다. 사용자에게는 짧은 평가, 근거 요약, caveat, 데이터 상태만 보여줍니다.

평가 입력:

- 관심도: 최근 언급량, 증가율, 반복 언급 여부, source 다양성
- 반응 방향: 기대, 우려, 혼조, 관망
- 주요 쟁점: 교통, 학군, 전세, 재건축, 청약, 대출, 공급, 정책 등
- 유사 과거: 비슷한 반응 패턴과 이후 실거래/전세/매물 흐름
- 최근 이슈: 뉴스, 컬럼, 정책, 개발, 금리, 대출, 교통 이슈 후보
- 시장 사실: 실거래가, 전세가, 매물량, 청약, 정책 데이터의 기준 시각

평가 출력:

- 대상: 지역 또는 단지
- 관심도 요약
- 반응 방향과 주요 쟁점
- 유사 과거 비교
- 최근 이슈 후보
- 시장 사실 요약과 `provider/asOf/stale`
- 종합 평가
- 근거 로그: 사용한 지표, 링크, snapshot id, 검색 provider, model version, prompt version

기록 후보:

- `targetType`, `targetId`
- `evaluatedAt`
- `summary`
- `evidence`
- `caveats`
- `source/asOf/stale`
- `strategyVersion` 또는 `evaluationVersion`
- `skipReason`

## 주요 데이터 도메인

- RealEstateRegion: 시/구/동/생활권 같은 지역
- RealEstateComplex: 아파트/주거 단지
- RealEstateTargetGraph: 지역, 생활권, 정책 영향권, 단지의 상하위/영향 관계
- RealEstateAlias: 공식명, 줄임말, 커뮤니티 별칭, 오타, 생활권 표현
- CrawlSource: 공공데이터, 공개 게시판, 카페, 뉴스/컬럼 source registry
- CommunityPost: 제한 저장된 게시글 메타데이터와 snippet
- RealEstateMention: 게시글 안에서 인식된 지역/단지 언급
- ReactionAnalysis: 언급별 반응 방향과 쟁점
- RealEstateReactionSnapshot: 지역/단지/window 단위 반응 지표
- RealEstateMarketFact: 실거래/전세/매물/정책/공급/뉴스 상태
- RealEstatePolicyEvent: 정책, 교통, 공급, 대출, 청약, 학군, 재건축 이벤트와 영향 범위
- RealEstateTimelineEvent: 반응, 뉴스, 정책, market fact를 시간순으로 묶은 이벤트
- EvidenceLog: 에이전트 평가와 근거 기록

## 명시적 제외 및 주의 사항

- 특정 매수, 매도, 청약, 대출 행동을 권유하지 않습니다.
- 가격 상승, 수익, 청약 성공을 단정하지 않습니다.
- 커뮤니티 로그인, CAPTCHA 회피, 프록시 우회, fingerprint 위장은 하지 않습니다.
- 원문 대량 저장과 외부 원문 재배포는 하지 않습니다.
- 작성자 개인별 성향 추적, 닉네임 랭킹, 개인 프로필 수집은 하지 않습니다.
- 내부 chain-of-thought는 저장하거나 노출하지 않고, 사용자용 짧은 근거만 저장합니다.

## 작업 영역과 정본 위치

- `realestate` 작업: 지역/단지, market fact, 정책 이벤트, alias 문서를 기준으로 합니다.
- `community` 작업: 공개 수집, source policy, 원문 제한 저장 문서를 기준으로 합니다.
- `indicator` 작업: 반응 방향, 쟁점 비율, snapshot, 표본 신뢰도 문서를 기준으로 합니다.
- `agent` 작업: 지역/단지 평가, 근거 로그, 유사 과거 비교 설명을 기준으로 합니다.
- `ui` 작업: `layers/ui` 문서와 화면별 Screen Brief를 기준으로 합니다.
- `ops` 작업: `layers/ops` 문서를 기준으로 합니다.
- 기존 금융 서비스 전용 영역은 active runtime에서 제거했고, 필요한 맥락은 git history와 결정 로그로만 확인합니다.
