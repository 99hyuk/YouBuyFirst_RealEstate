# 커뮤니티 반응 신호 가이드

이 문서는 너나사 (YouBuyFirst)에서 커뮤니티 글을 분석할 때 쓰는 제품 용어와 수집 전략을 정리합니다.

## 용어 기준

너나사는 `감성`이라는 표현을 대표 용어로 쓰지 않습니다. 서비스가 보여주려는 것은 막연한 기분 분석이 아니라, 커뮤니티 글에서 관찰되는 투자 관련 반응을 데이터화한 신호입니다.

대표 용어:

- `커뮤니티 반응 신호`: 제품 전체에서 쓰는 상위 개념
- `반응 지표`: 화면과 API 설명에서 쓰는 집계 지표
- `반응 모멘텀`: 언급량과 반응 강도의 증가 흐름
- `반응 방향`: 단일 게시글 안에서 특정 종목이 어떤 방향으로 언급되는지 나타내는 값

반응 방향 표시값:

| 내부 값 | 사용자 표시 | 의미 |
| --- | --- | --- |
| `bullish` | 낙관 | 상승 기대, 호재 기대, 긍정적 전망에 가까운 반응 |
| `bearish` | 비관 | 하락 우려, 악재 우려, 부정적 전망에 가까운 반응 |
| `neutral` | 중립 | 명확한 방향이 없거나 정보 공유에 가까운 반응 |

향후 코드/DB 이름을 정리할 때는 `stance`보다 `reactionDirection`을 우선 후보로 둡니다. `stance`는 분석가 관점의 단어라 짧지만, 새 작업자가 바로 이해하기에는 `reactionDirection`이 더 직관적입니다.

## 피해야 할 표현

- `감성 분석`: 필요하면 legacy 구현이나 과거 문서 설명에서만 사용합니다.
- `긍정/부정`: 투자 맥락에서는 감정 평가처럼 보이므로 `낙관/비관`을 우선합니다.
- `매수/매도 신호`: 투자 자문처럼 보일 수 있으므로 사용자 화면에서는 쓰지 않습니다.
- `추천`, `수익 보장`, `따라 사기`: 제품 목적과 법적 리스크 기준에 맞지 않습니다.

## 소스 구조별 수집 전략

모든 커뮤니티를 같은 방식으로 수집하지 않습니다. 소스가 시장 전체 이야기를 섞어 보여주는지, 종목별 방으로 나뉘는지에 따라 역할을 나눕니다.

### 일반 게시판형

대상 예시:

- 에펨코리아 주식 게시판
- 디시인사이드 주식갤러리

역할:

- 시장 전체 반응 흐름 파악
- 새로 언급이 튀는 종목 발견
- 라이징 스타 후보와 테마 후보 생성

수집 방식:

- 최근 글 목록을 30분 단위로 수집합니다.
- 제목과 제한 snippet에서 종목명, 티커, 별칭을 인식합니다.
- 언급량 증가율, 키워드 변화, 반응 방향 분포를 이용해 후보 종목을 만듭니다.

### 종목 게시판형

대상 예시:

- 네이버 종토방
- 토스 종목 커뮤니티

역할:

- 특정 종목에 대한 깊은 반응 확인
- 일반 게시판형에서 발견한 후보 종목 검증
- 종목별 보유자 불안, 기대, 이슈 반응 보강

수집 방식:

- 모든 종목을 30분마다 전수 수집하지 않습니다.
- `CrawlTarget` 큐로 우선순위를 관리합니다.
- 우선순위는 일반 게시판 언급 증가, 거래대금, 급등락, 사용자 관심 종목, 최근 활동량, 실패 backoff를 함께 반영합니다.

## 신호 사용 방식

| 목적 | 우선 소스 | 보조 소스 |
| --- | --- | --- |
| 시장 전체 반응 | 일반 게시판형 | 시세/거래량 |
| 라이징 스타 후보 | 일반 게시판형 | 종목 게시판형, 시세/거래량 |
| 종목별 반응 상세 | 종목 게시판형 + 일반 게시판형 | AI 재서술 근거 |
| 커뮤니티별 성과 비교 | 소스별 반응 지표 | 이후 가격 변화 |

이 기준은 “수집 가능한 모든 글을 긁는 서비스”가 아니라, 투자 참고에 필요한 커뮤니티 반응을 보수적으로 추출하는 제품을 만들기 위한 경계입니다.

## 트랙별 책임

- `crawl`: 소스 구조를 구분하고, 안전한 방식으로 제한 원문과 메타데이터를 수집합니다.
- `data`: 종목 인식, 반응 방향 분류, 반응 지표와 30분 집계를 만듭니다.
- `market`: 반응 지표와 함께 볼 시세/호가 데이터를 제공합니다.
- `agent`: 반응 지표와 시세를 읽어 전략 판단과 성과 비교를 만듭니다.
- `front`: 반응 신호를 투자 참고 화면에서 이해하기 쉽게 보여줍니다.
- `ops`: 용어, 문서, Notion, PR 기록이 같은 기준을 따르는지 점검합니다.
