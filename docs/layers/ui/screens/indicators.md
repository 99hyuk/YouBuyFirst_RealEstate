# 주요 일정 화면

## Route

- Parent: root
- Overview route: `/indicators`
- Legacy detail routes:
  - `/indicators/price-transaction`
  - `/indicators/supply-demand`
  - `/indicators/demand-mood`
  - `/indicators/macro-finance`

상단 내비게이션 노출명은 `주요 일정`입니다. 기존 지표 상세 route는 외부 링크나 과거 북마크가 깨지지 않도록 같은 일정 화면을 렌더링합니다.

## 화면 목적

기존의 가격·거래량/공급·수급/심리/금융 지표 허브는 폐기합니다. 이 화면은 지역별 일정이 아니라 부동산 시장을 볼 때 반복적으로 확인해야 하는 전국 단위 공식 발표, 통계 공개, 정책·금리·청약 일정을 캘린더 형태로 보여줍니다. 사용자는 날짜 칸의 일정 줄을 눌러 아래 요약 행으로 이동하고, 요약 행에서 관련 공식 출처로 이동하며, 어떤 데이터를 언제 확인해야 하는지 빠르게 판단합니다.

## 현재 섹션

- 일정 히어로: 지역 분석/마이페이지 화면의 공통 헤더 타이포와 간격을 따르는 화면 제목과 페이지 용도 설명
- 월간 캘린더: 오른쪽 upcoming 패널 없이 화면 폭을 넓게 사용하며, 월 제목 양옆 화살표로 이전/다음 달을 이동합니다. 월을 바꾸면 `GET /api/realestate/market-data-schedules?month=YYYY-MM`를 다시 조회하고, 날짜 칸과 하단 일정 요약 목록을 같은 월 데이터로 함께 갱신합니다. 날짜 칸에는 `제목 · 한 문장 요약`을 흰 배경, 짙은 본문, 얇은 tone 표시선의 고대비 일정 줄로 표시하고 캘린더 아래에도 같은 일정의 상세 요약 행을 표시합니다. 헤더 오른쪽에는 가격·공시, 거래현황, 공급, 금융, 정책, 청약 색상 범례를 둡니다. 날짜 칸 일정 줄은 외부 링크가 아니라 같은 일정의 하단 요약 행으로 이동하는 요청 버튼이며, 이동된 요약 행에는 짧은 요청 확인 애니메이션을 적용합니다. 하단 요약 행은 실제 공표 상세 URL, 청약 모집공고 상세, 통계 세부 화면, 통화정책 일정 화면처럼 유효한 URL이 있을 때 관련 일정 사이트로 이동합니다. 날짜 칸 일정 줄의 배경은 상태 의미로 쓰지 않고, `공표 확인/공식 일정` 상태는 하단 상세 요약 행에서만 읽게 합니다. 좁은 화면에서는 7열을 억지로 압축하지 않고 캘린더 본문만 가로 스크롤해 일정 row 폭을 보존합니다.
- 공식 출처: 화면 하단 오른쪽 1/3 폭의 보조 링크 묶음으로 제공하며, 제목과 카드를 오른쪽 기준에 맞추고 설명문 없이 출처명과 분류만 3열 compact grid로 작게 표시

## 일정 항목 기준

- 일정은 투자 행동을 지시하지 않고 `확인`, `점검`, `공개`, `발표`처럼 관찰형 동사로 표현합니다.
- 공식 원천에서 해당 월 날짜가 확인되지 않은 반복 점검/체크리스트 row는 캘린더 일정으로 만들지 않습니다.
- 실제 공표 목록에서 확인한 row는 `dataStatus=published`, 통계누리 다음공표일·한국은행 회의일처럼 공식 일정 페이지에서 확인한 row는 `dataStatus=scheduled`, 출처 확인 실패는 `dataStatus=source_error`로 구분합니다.
- 각 일정은 `provider`, `asOf`, `sourceUrl`, `scheduleDate`, `stale`, `lastCheckedAt`, `status`를 함께 가져야 합니다.
- 날짜 칸 안의 일정 줄과 하단 요약 행은 한 문장 summary를 포함합니다.
- 날짜 칸 일정 줄은 외부 링크를 열지 않고 하단 요약 행으로 이동합니다. 링크는 하단 요약 행에서만 열며, 대표 홈페이지나 일반 목록이 아니라 실제 공표 상세 URL, 청약 모집공고 상세, 통계 세부 화면, 통화정책 일정 화면처럼 해당 일정 맥락을 바로 확인할 수 있는 URL일 때만 씁니다.

## 공식 출처

- 한국부동산원 R-ONE: 가격지수·거래현황 공표일정, 주간·월간 통계 공표자료
- 국토교통부 실거래가 공개시스템: 매매·전월세 실거래 freshness/status 확인. 고정 캘린더 row로 만들지 않습니다.
- 국토교통 통계누리: 미분양, 인허가, 착공, 준공 등 공급 통계
- 한국은행: 기준금리, 통화정책방향 결정회의, 자금순환·통화유동성·가중평균금리·소비심리·경제심리 등 월간통계 공표일정
- 청약Home: 모집공고, 청약 접수 시작/마감, 당첨자 발표
- 국토교통부: 정책, 공급, 교통, 정비사업 보도자료
- HUG 주택도시보증공사: 민간아파트 분양가격, 초기분양률, 분양시장 공표자료
- LH청약플러스: 공공분양, 분양주택, 공공임대리츠처럼 시장 관찰 신호가 큰 공급 공고. 예비입주자 모집, 정정공고, 행복주택·국민임대·영구임대·매입임대 일반 row는 캘린더 과밀을 막기 위해 제외합니다.
- 서울주거포털 SH: 공공임대 입주자모집공고
- 경기주택도시공사 GH: 분양·임대 공고
- 인천도시공사 iH: 분양·임대 입주자모집공고
- 금융위원회: 부동산 금융정책, 가계부채, 주택담보대출 관련 보도자료
- 부동산공시가격 알리미: 공동주택가격 열람, 결정·공시, 이의신청 공지

## 상태와 빈 화면

- loading: 캘린더 frame과 skeleton 일정 줄을 먼저 보여줍니다.
- empty: 해당 월 일정이 없으면 공식 출처 링크를 유지하고 `일정 수집 전/insufficient`를 표시합니다.
- stale: 일정은 노출하되 마지막 확인 시각과 provider를 함께 표시합니다.
- error: 캘린더는 유지하고 출처별 오류를 분리합니다.

## API

| 필드 | 소유 도메인/layer | 설명 |
| --- | --- | --- |
| `scheduleEvents` | content/realestate | 캘린더에 표시할 공식 일정 |
| `sourceLinks` | content/realestate | 공식 출처 링크 |
| `lastCheckedAt` | backend | provider별 마지막 확인 시각 |
| `stale` | backend | 일정 갱신 지연 여부 |

현재 endpoint:

- `GET /api/realestate/market-data-schedules?month=YYYY-MM`
- `GET /api/realestate/batch-updates/stream`

`marketDataScheduleRefreshJob`이 `market_data_sources`와 `market_data_schedules`를 갱신합니다. 배치는 뉴스 검색 크롤링이 아니라 한국부동산원 R-ONE 보도자료·통계공표일정, 국토부 보도자료, 청약Home APT 모집공고·접수·당첨 일정, HUG 분양시장 공표자료, LH/SH/GH/iH 공공 공급 공고, 금융위원회 부동산 금융정책 보도자료, 부동산공시가격 알리미 공동주택가격 공지, 통계누리 다음공표일, 한국은행 통화정책방향 결정회의·월간통계 공표일정처럼 공식 원천에서 실제 날짜가 확인된 row만 저장합니다. 한국은행 월간통계는 부동산 투자 환경과 연결되는 자금순환, 통화 및 유동성, 국민대차대조표, 생산자물가지수, 국내총생산, 금융기관 가중평균금리, 소비자동향조사, 기업경기조사·경제심리지수만 금융 일정으로 표시합니다. 실거래 공개시스템처럼 상시 공개 성격의 데이터는 고정 캘린더 row가 아니라 freshness/status 확인 대상으로 둡니다. 출처 URL 확인 결과로 `확인 완료`, `확인 필요`, `dataStatus`, `stale`, `lastCheckedAt`를 갱신하되, 출처 확인만 성공했다고 일반 점검 일정을 만들지는 않습니다. 같은 월을 다시 갱신할 때는 이번 배치에서 확인된 row로 월별 일정을 replace해 사라진 공고나 잘못된 과거 row가 계속 남지 않게 합니다.

배치가 저장을 끝내면 SSE push 채널이 `realestate-batch-update` 이벤트의 `topic=market-data-schedules`, `month=YYYY-MM`을 발행합니다. 프론트는 이벤트 payload를 일정 데이터로 직접 쓰지 않고, 현재 열려 있는 월과 일치할 때 `GET /api/realestate/market-data-schedules?month=YYYY-MM`를 다시 조회합니다.

기존 `/api/realestate/indicators`, `/api/realestate/indicators/:category`, `/api/realestate/indicators/anomalies`는 이 화면의 필수 입력이 아닙니다. 지표 값 자체는 대시보드 오른쪽 지표 탭이나 대상 상세 화면에서 다룹니다.

## 기획 확인 필요

- 한국은행 결정문/의사록, 통계누리 공표 완료 row, 금감원/기재부 부동산 금융정책 공표는 추가 수집 후보입니다.
- 정책·보도자료는 전체를 가져오면 노이즈가 커서, 부동산/공급/교통/대출 키워드 필터 기준을 계속 조정해야 합니다.
