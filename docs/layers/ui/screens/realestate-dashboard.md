# 부동산 대시보드

## Route

- Parent: root app
- Route 후보: `/dashboard`, `/realestate`
- Child screens: `realestate-target-detail`, `transaction-explorer`, `indicators`

## 화면 목적

사용자가 접속 직후 오늘의 핵심 부동산 상황을 먼저 이해하고, 이어서 어느 지역의 실거래·전세·공급·정책 흐름을 확인해야 하는지 빠르게 판단합니다. 공개 반응 데이터는 있으면 보조 신호로만 붙이고, 대시보드의 핵심 순위나 정체성으로 세우지 않습니다.

## 현재 섹션

- 상단 검색: 검색창 위에 `너나사 YouBuyFirst` 브랜드 타이틀을 크게 노출하고, 제목 아래 태그라인은 `당신을 위한 부동산 인사이트, 너나사`로 둬서 부동산 서비스 맥락을 간접적이지만 또렷하게 알림. 상단 영문 보조 라벨은 태그라인과 중복되므로 노출하지 않음. 지역, 단지, 법정동, 키워드 검색 진입점을 둠
- Daily briefing 3줄 브리핑: 특정 지역 차트나 바로가기 카드 묶음이 아니라 저장된 일일 브리핑의 `summaryHeadlines[3]`만 짧은 명사형 헤드라인으로 표시. 대시보드도 `/realestate/daily-briefing` 상세의 3개 헤드라인 카드 디자인을 공유하되, 대시보드에서는 3개 요약을 세로로 쌓고 오른쪽 `지역·단지 반응 한눈에` 박스의 상하 높이에 맞춰 늘림. 3줄 브리핑 박스의 좌우 폭은 하단 `실시간 뉴스` 카드와 끝선이 맞도록 같은 그리드 간격을 사용함. CTA는 브랜드 주황 배경보다 어두운 고대비 리서치 카드 톤으로 두고, `자세한 분석 보러가기` 오른쪽 중앙에 밝은 원형 화살표 아이콘을 둠. 3개 요약 카드의 모서리는 대시보드에서만 조금 더 둥글게, 역할 라벨과 헤드라인은 여백에 맞춰 크게 표시함. 화면에는 `24시간`, `7일`, `1개월`, `3개월` 같은 구간 제목을 노출하지 않음
- 일일 브리핑 상세: `/realestate/daily-briefing`은 줄글 본문 화면이 아니라 건조한 금융권 리서치 브리프에 가까운 브리핑 보드로 구성함. 상단 기준 정보, 3개 헤드라인 카드, 핵심 흐름 리드/해설 문단, 단기 지표 기반 주목 지역 ledger, 간결한 시장 변수 list, 관련 뉴스·리포트 ledger를 분리해 보여줌. 실제 `sourceItems`가 있으면 출처 라벨과 원문 근거 링크로 표시하고, 없을 때는 본문을 반복하지 않고 뉴스룸, 정책·통계 리포트, 주요 일정, 전국 지역 지도 같은 내부 확인 화면으로 연결함
- 확인할 지역/생활권 후보. 기대/우려/중립 아이콘 범례처럼 카드 내용을 반복 설명하는 보조 UI는 두지 않고, `지역·단지 반응 한눈에` 헤더에는 기간 탭이나 상세 바로가기 액션을 노출하지 않음. 반응 카드 헤더와 첫 카드 시작선은 왼쪽 `Daily briefing` 3줄 브리핑 헤더/카드 시작선과 같은 높이를 유지함
- 실거래·전세·공급 변화 요약
- 실시간 이슈, 뉴스, 컬럼 링크
- 실거래/전세/매물/정책 데이터 상태 badge
- 공개 지연, provider/asOf, 데이터 부족 rail
- 공통 footer: 커뮤니티 반응 중심 설명이 아니라 실거래·전세 흐름, 지도 기반 지역 변화, 주요 일정, 뉴스·리포트 이슈를 정리하는 부동산 인사이트 서비스로 설명
- 오른쪽 빠른 패널: `반응/지표/관심` 탭 전환과 `라이징 스타` 보조 카드를 두지 않고, 첫 화면에서는 `라이브 패널 · 주간 가격 흐름`을 본문과 같은 카드 안의 상단 헤더로 합쳐 유지. 헤더는 작은 라벨이 아니라 카드 상단을 꽉 채우는 진한 오렌지색 면으로 두어 즉시 눈에 들어오게 함. 본문 카드는 가운데 `지역·단지 반응 한눈에` 슬라이드 카드의 윗단과 같은 높이에서 시작하도록 배치함. 헤더와 상승 영역 사이에는 별도 gap을 두지 않고 바로 맞닿게 함. 본문에는 `언급`, `열기`, `시장`, `상승률`, `표본`, `데이터 근거` 같은 보조 지표 묶음을 쓰지 않고, 위 절반에는 `이번주 가장 많이 상승한 지역`, 아래 절반에는 `이번주 가장 많이 하락한 지역`을 표시함. 두 영역은 절반씩 나누고 기존 연한 실선으로 구분하며, 각 영역에서는 `이번주...지역` 라벨을 가장 위에 두고 지역명을 그 아래 왼쪽에 둔 뒤, 상승/하락 수치는 오른쪽 아래에 둬 대각선으로 읽히게 함. `최근 1주 · 기준일` 메타도 카드 하단 오른쪽 끝에 둠. 두 영역은 폐기된 target 상세가 아니라 `/realestate/map/:regionId?selectedTargetId=:targetId&selectedRegionCode=:geometryCode&period=week`로 이동해 지역 분석 지도의 해당 시군구 리포트 패널을 바로 열어야 함. 지표, 실거래, 저장 지역 관리는 각 전용 화면으로 이동해 확인함

## 인사이트 지표 방향

기존 금융 화면의 지수/등락률 패턴을 그대로 가져오면 부동산에서는 의미가 흐려질 수 있습니다. 부동산은 거래 빈도가 낮고 신고·공개 지연이 있으며, 지역·단지·면적·연식에 따라 같은 기간의 가격 신호가 다르게 보입니다. 따라서 첫 화면은 지표 나열보다 "지금 확인할 시장 사실 묶음"으로 둡니다.

- 가격 흐름: 실거래가, 전세가, 공표 지수의 방향과 `asOf`, 표본 수, stale 여부를 함께 표시
- 거래 강도: 거래 건수, 신고 건수, 매물 소진 체감, 거래량 부족 caveat 표시
- 전세 압력: 전세가 변화, 전월세 비중, 전세 매물 감소/증가 후보
- 공급/청약: 입주 예정, 분양/청약 일정, 경쟁률, 미분양 후보 데이터
- 정책/교통 이벤트: 대출, 세제, 정비사업, 철도/도로 발표를 원인 확정이 아닌 context로 표시
- 보조 공개 반응: 기대/우려/중립, 언급량, 별칭 확산, source skew를 가격 사실과 분리 표시하되 핵심 지표로 승격하지 않음
- 핵심 지역 흐름: 마포구, 성수동, 동탄역권, 송도처럼 사용자가 보는 지역 단위의 월/3개월/6개월/년 변화율로 표시

카드 라벨은 `상승률`, `강세` 같은 결론형 표현보다 `실거래 공개`, `전세 압력`, `거래 강도`, `정책 이벤트`, `주요 일정`처럼 사용자가 직접 판단할 수 있는 이름을 우선합니다.

## 상태와 빈 화면

- loading: market summary skeleton, timeline skeleton, stale badge placeholder
- empty: 확인할 시장 fact 없음, provider 상태와 mock 여부 표시
- error: 수집/API 실패 영역별 표시
- stale/mock: provider, `asOf`, `stale`, `mock` badge를 카드마다 표시

## API 후보

| 필드 | 소유 도메인/layer | 설명 |
| --- | --- | --- |
| `targets[]` | realestate/backend | 확인할 지역/단지 후보 |
| `targets[].marketFacts` | realestate | 실거래/전세/매물/정책 fact와 provider/asOf/stale |
| `targets[].scheduleSummary` | realestate/backend | 공식 통계, 청약, 정책 일정 요약 |
| `issueFeed[]` | content/realestate | 뉴스/컬럼/보조 공개 원문 링크 |
| `dailyBriefing` | ui/backend/agent | `GET /api/realestate/daily-briefings/latest` 최신 저장 브리핑. 대시보드는 `summaryHeadlines[3]`, 상세 화면은 본문 섹션을 공유 |

현재 구현:

- `GET /api/realestate/dashboard/market-summary` 응답은 대시보드의 시장 요약과 오른쪽 `인사이트` 탭에 우선 표시합니다. `/indicators`는 지표 상세가 아니라 공식 발표 일정 화면으로 둡니다. API가 비거나 실패하면 fixture 지표를 섞지 않고 `수집 전/insufficient` 또는 오류 상태를 표시합니다.
- 2026-06-16 MVP 기준 `dashboard/market-summary`는 한국부동산원 R-ONE 전국 주요 지표(`매매가격지수 변동률`, `전세가격지수 변동률`, `아파트 매매거래호수`)를 우선 표시할 수 있습니다. 이 값은 전국 배경 지표이며, 지역별 가격 fact처럼 가중하거나 단정하지 않습니다.
- `GET /api/realestate/transactions/summary?windowMinutes=10080&limit=10` 응답을 관심 drawer, 거래 drawer, 지역 시장 흐름 카드에 우선 표시합니다. 검색창 주변에는 반응 필터, 언급 합계/지연 지표/관찰 수 메타 칩, "7일 · 반응 데이터 반영" 같은 기간·상태 eyebrow, "특정 지역에 반응이 가장 많이 몰렸다" 같은 단정형 자동 헤드라인, 임시 gauge를 만들지 않습니다. API가 비거나 실패하면 fixture를 섞지 않고 `수집 전/insufficient` 또는 오류 상태를 표시합니다.
- 대시보드 좌측 첫 블록의 `Daily briefing` 3줄 브리핑은 `GET /api/realestate/daily-briefings/latest` 응답의 `summaryHeadlines[3]`만 표시합니다. API가 비면 mock 문장을 조합하지 않고 `브리핑 생성 대기` 상태로 둡니다. 대시보드 요약 3개는 상세 브리핑 페이지의 `daily-briefing-headline-card` 디자인을 공유하되 세로 3줄로 배치하고, 카드 묶음 전체 높이는 오른쪽 반응 패널과 맞춥니다. `자세한 분석 보러가기`는 오른쪽 화살표 CTA와 함께 `/realestate/daily-briefing`으로 이동합니다.
- `GET /api/realestate/map/layers?layerType=sigungu` 응답의 `week` 기간은 오른쪽 빠른 패널의 주간 상승 1위/하락 1위 세부지역에만 사용합니다. 세부지역별 비교/지도 탐색은 `/realestate/map`에서 다룹니다.
- `GET /api/realestate/newsroom?feed=all&page=1&pageSize=40` 응답을 대시보드 뉴스/리포트/영상/커뮤니티 링크 카드에 우선 표시합니다. API가 비거나 실패하면 mock feed 대신 수집 전/오류 상태를 표시합니다.
- SerpApi/content 후보는 화면에서 임의 복제하지 않습니다. 리포트는 KB금융/KB Think, 하나금융연구소, 우리금융경영연구소, 주택금융연구원/HF, 한국금융연구원/KIF, 건설산업연구원, 국토연구원/K-REMAP, 한국은행, HUG, 한국부동산원 같은 금융·연구·공공기관 도메인만 `정책·통계 리포트`로 분류하고, 언론사가 보고서 발간 소식을 다룬 기사는 `실시간 뉴스`로 둡니다. 배치는 기관군별 source로 나뉘어 한 기관 결과가 카드 전체를 독점하지 않게 합니다.
- 영상 후보는 집코노미, 매부리TV, 부읽남TV, 삼프로TV, 스마트튜브, 김작가TV처럼 부동산 분석·토론·인터뷰 맥락이 있는 YouTube 채널 RSS에서만 가져옵니다. Shorts URL, 600초 미만 영상, 조회수 10,000회 미만 영상, 검색어에 "영상"이 들어간 일반 기사, 채용공고, 공유 래퍼 페이지, 액션형 추천 제목은 영상 카드에 넣지 않습니다. 공개 YouTube 상세 메타의 `keywords`와 Atom RSS의 `media:description`/`media:statistics`가 부동산 주제·조회수 근거를 줄 때 제목 주제어 부족을 보완하는 근거로 사용합니다.
- 블로그/커뮤니티 링크는 단순 블로그 검색이 아니라 컬럼형 원문 출처를 우선합니다. 배치 기준은 KB Think 부동산 컬럼, 빠숑 네이버 블로그 RSS, 빌딩의 정석 네이버 RSS, Tistory 시장분석 RSS처럼 공개 RSS/XML로 확인되는 출처를 두고, 주식·강의·모집공고·매물·경매물건·홍보·액션형 추천 문구 같은 잡음은 제외합니다.
- 뉴스/리포트/영상/블로그 4종 카드는 수동 큐레이션만으로 채우지 않고 `newsroomContentRefreshJob`이 2시간마다 수집한 feed별 최신 통과 항목을 우선 표시합니다. 주요일정은 별도 `marketDataScheduleRefreshJob`이 하루 한 번 갱신합니다.
- API가 비어 있거나 실패하면 `수집 대기`, `데이터 없음`, `공공데이터 대기` 상태로 표시하고, fixture 값을 실제 데이터처럼 보이게 대체하지 않습니다.
- 요약 카드의 `changePct`가 없으면 `0%`로 보정하지 않고 `최신` 또는 `변화율 없음`으로 표시합니다.
- row와 카드에는 `provider`, `observedAt`, `asOf`, `stale/dataStatus`를 함께 노출합니다.
