# 부동산 대시보드

## Route

- Parent: root app
- Route 후보: `/dashboard`, `/realestate`
- Child screens: `realestate-target-detail`, `transaction-explorer`, `indicators`

## 화면 목적

사용자가 접속 직후 오늘의 핵심 부동산 상황을 먼저 이해하고, 이어서 어느 지역의 실거래·전세·공급·정책 흐름을 확인해야 하는지 빠르게 판단합니다. 공개 반응 데이터는 있으면 보조 신호로만 붙이고, 대시보드의 핵심 순위나 정체성으로 세우지 않습니다.

## 현재 섹션

- 상단 검색: 검색창 위에 `너나사 YouBuyFirst` 브랜드 타이틀을 크게 노출하고, 제목 아래 태그라인은 `당신을 위한 부동산 인사이트, 너나사`로 둬서 부동산 서비스 맥락을 간접적이지만 또렷하게 알림. 상단 영문 보조 라벨은 태그라인과 중복되므로 노출하지 않음. 지역, 단지, 법정동 검색 입력을 실제 자동완성으로 연결하고, 검색어는 2글자 이상이 되는 즉시 `GET /api/realestate/targets/search?mode=autocomplete&limit=8`을 호출한다. 결과는 검색창 아래 리스트로 표시하며, 결과를 한 번 누르면 리스트를 닫고 바로 이동한다. 시도/시군구/생활권은 지역 분석 지도 리포트 패널이 선택된 상태로, 단지와 동 단위 `eupmyeondong`은 실거래 탐색 화면의 필터와 첫 매칭 상세가 열린 상태로 보낸다. 검색 결과 수나 외부 검색 노출량을 관심도 지표처럼 표시하지 않음
- 검색 자동완성 보정: 한글 IME에서 조합 완료를 기다리지 않고, 두 번째 글자가 입력창 값에 보이는 순간 2글자 이상이면 즉시 호출한다. `regionLevel=eupmyeondong` 결과는 지역 분석이 아니라 `/realestate/transactions?region=<시군구 5자리>&q=<동명>`로 보내고, 실거래 화면은 route `q`와 일치하는 첫 결과를 상세 선택 상태로 열며 지도 중심을 해당 단지 좌표로 둔다.
- Daily briefing 3줄 브리핑: 특정 지역 차트나 바로가기 카드 묶음이 아니라 저장된 일일 브리핑의 `summaryHeadlines[3]`만 짧은 명사형 헤드라인으로 표시. 대시보드도 `/realestate/daily-briefing` 상세의 3개 헤드라인 카드 디자인을 공유하되, 대시보드에서는 3개 요약을 세로로 쌓고 오른쪽 `지역·단지 반응 한눈에` 박스의 상하 높이에 맞춰 늘림. 3줄 브리핑 박스의 좌우 폭은 하단 `실시간 뉴스` 카드와 끝선이 맞도록 같은 그리드 간격을 사용함. CTA는 브랜드 주황 배경보다 어두운 고대비 리서치 카드 톤으로 두고, `자세한 분석 보러가기` 오른쪽 중앙에 밝은 원형 화살표 아이콘을 둠. 3개 요약 카드의 모서리는 대시보드에서만 조금 더 둥글게, 역할 라벨과 헤드라인은 여백에 맞춰 크게 표시함. 화면에는 `24시간`, `7일`, `1개월`, `3개월` 같은 구간 제목을 노출하지 않음
- 일일 브리핑 상세: `/realestate/daily-briefing`은 줄글 본문 화면이 아니라 건조한 금융권 리서치 브리프에 가까운 브리핑 보드로 구성함. 상단 기준 정보, 3개 헤드라인 카드, 핵심 흐름 리드/해설 문단, 단기 지표 기반 주목 지역 ledger, 간결한 시장 변수 list, 관련 뉴스·리포트 ledger를 분리해 보여줌. 실제 `sourceItems`가 있으면 출처 라벨과 원문 근거 링크로 표시하고, 없을 때는 본문을 반복하지 않고 뉴스룸, 정책·통계 리포트, 주요 일정, 전국 지역 지도 같은 내부 확인 화면으로 연결함
- 확인할 지역/생활권 후보. 기대/우려/중립 아이콘 범례처럼 카드 내용을 반복 설명하는 보조 UI는 두지 않고, `지역·단지 반응 한눈에` 헤더에는 기간 탭이나 상세 바로가기 액션을 노출하지 않음. 반응 카드 헤더와 첫 카드 시작선은 왼쪽 `Daily briefing` 3줄 브리핑 헤더/카드 시작선과 같은 높이를 유지함
- `지역·단지 반응 한눈에` 카드 본문은 `GET /api/realestate/map/layers?layerType=sigungu`의 주간 상승률 양수 TOP5만 슬라이드로 표시하고, 각 카드 오른쪽 위에는 `주간 +x.xx%`를 한 줄 배지로 둠. 카드의 `기대`/`우려` 영역은 단어 구름을 쓰지 않고 같은 target의 최신 지역 리포트 `expectationPoints`, `concernPoints` 문장을 최대 3개씩 그대로 표시함. 문장은 줄바꿈되며, 각 문장 행은 작은 정보 카드처럼 세로 공간을 고르게 채워 빈 여백이 크게 남지 않아야 함. 카드 전체를 클릭하면 `/realestate/map/:regionId?selectedTargetId=:targetId&selectedRegionCode=:geometryCode&period=week`로 이동해 지역 분석 지도 안의 해당 시군구 리포트 패널을 바로 열어야 함. 카드 우상단 언급 배지, 지역/단지·생활권 `열기` 메타, 기대/우려 관련 링크 묶음, 반응 비율 바는 노출하지 않음
- 실거래·전세·공급 변화 요약
- 실시간 이슈, 뉴스, 컬럼 링크
- 실거래/전세/매물/정책 데이터 상태 badge
- 공개 지연, provider/asOf, 데이터 부족 rail
- 공통 footer: 커뮤니티 반응 중심 설명이 아니라 실거래·전세 흐름, 지도 기반 지역 변화, 주요 일정, 뉴스·리포트 이슈를 정리하는 부동산 인사이트 서비스로 설명
- 오른쪽 빠른 패널: `반응/지표/관심` 탭 전환과 `라이징 스타` 보조 카드를 두지 않고, 첫 화면에서는 `라이브 패널 · 주간 가격 흐름`을 본문과 같은 카드 안의 상단 헤더로 합쳐 유지. 헤더는 작은 라벨이 아니라 카드 상단을 꽉 채우는 진한 오렌지색 면으로 두어 즉시 눈에 들어오게 함. 본문 카드는 가운데 `지역·단지 반응 한눈에` 슬라이드 카드의 윗단과 같은 높이에서 시작하도록 배치함. 헤더와 상승 영역 사이에는 별도 gap을 두지 않고 바로 맞닿게 함. 본문에는 `언급`, `열기`, `시장`, `상승률`, `표본`, `데이터 근거` 같은 보조 지표 묶음을 쓰지 않고, 위 절반에는 `이번주 가장 많이 상승한 지역`, 아래 절반에는 `이번주 가장 많이 하락한 지역`을 표시함. 두 영역은 절반씩 나누고 기존 연한 실선으로 구분하며, 각 영역에서는 `이번주...지역` 라벨을 가장 위에 두고 지역명을 그 아래 왼쪽에 둔 뒤, 상승/하락 수치는 오른쪽 아래에 둬 대각선으로 읽히게 함. `최근 1주 · 기준일` 메타도 카드 하단 오른쪽 끝에 둠. 두 영역은 폐기된 target 상세가 아니라 `/realestate/map/:regionId?selectedTargetId=:targetId&selectedRegionCode=:geometryCode&period=week`로 이동해 지역 분석 지도의 해당 시군구 리포트 패널을 바로 열어야 함. 지표와 실거래는 각 전용 화면으로 이동해 확인하고, 저장 지역 전용 화면은 두지 않음
- 공통 오른쪽 고정 rail: `관심`, `최근 본`, `채팅` 순서로만 둔다. 기존 `반응`, `실시간` 탭은 제거하고, 채팅 탭은 로그인 사용자와 비로그인 게스트가 닉네임으로 참여하는 좁은 피드 UI를 제공한다. 사이트 첫 진입 기본값은 채팅 패널이 열린 상태이며, 이때 최근 메시지 조회, SSE 연결, `/api/chat/presence` heartbeat를 바로 시작한다. 사용자가 패널을 닫거나 `관심`/`최근 본` rail로 전환하면 채팅 heartbeat를 멈춘다. 채팅방 내부에는 `전체/채팅/뉴스` 같은 보조 필터 탭을 두지 않고, 메시지 닉네임 옆에는 채팅/뉴스 같은 텍스트 badge를 반복 표시하지 않는다. 채팅 상단의 참여자 수는 버튼이 아니라 초록 점과 숫자가 같은 줄에 놓이는 상태 표시값으로 왼쪽에 두고, 값은 채팅 패널을 연 로그인/게스트 세션이 `/api/chat/presence`로 보낸 최근 45초 heartbeat 기준 `activeSessionCount`다. `A-`/`A+` 메시지 글자 크기와 닉네임은 오른쪽 끝 액션 묶음으로 정렬한다. 로그인 사용자는 `displayName` 텍스트를 그대로 보여주고 이 텍스트를 클릭하면 `닉네임 변경` 제목이 있는 작은 팝업에서 채팅 닉네임을 로컬로 수정할 수 있다. 로그인/게스트 모두 닉네임 문자열 해시 기반의 고정 색상 팔레트를 사용하며, 같은 닉네임은 같은 색상으로 표시한다. 가입자 작성 메시지는 그 위에 천천히 흐르는 얇은 하이라이트 띠 오버레이와 작은 인증 마크를 추가 표시한다. 비로그인 사용자는 채팅 목록을 블러/모자이크 오버레이로 본 뒤 `로그인 하기` 또는 `비로그인으로 참여하기`를 선택하고, 게스트 닉네임 입력 후 인증마크와 그라데이션 없이 채팅할 수 있다. `닉네임 설정` 문구 버튼, 톱니, 팝업, 닫기 버튼은 두지 않는다. 채팅 패널 배경은 본문 사이트 배경과 구분되는 블루그레이 톤으로 분리하고, 본문과의 경계에는 두꺼운 왼쪽 선과 그림자를 둔다. 내가 쓴 메시지는 오른쪽 정렬의 옅은 배경 말풍선으로 구분한다. 채팅 메시지는 `POST /api/chat/messages`로 서버 DB에 저장하고, 패널이 열릴 때 `GET /api/chat/messages?limit=50`으로 최근 메시지를 읽은 뒤 `/api/chat/messages/stream` SSE(EventSource)로 새 메시지를 즉시 반영한다. 5초 재조회는 stream 끊김을 보완하는 안전망으로 유지한다. 지역 분석/실거래 첨부는 메시지의 `attachment`로 저장하며, 첨부 카드는 `종류+제목 / 필요할 때만 보이는 설명 / 기간+등락률`로 압축한다. 실거래는 대상 설명을 유지해 3줄, 지역 분석은 설명이 기간과 중복되면 숨겨 2줄로 보이고, 카드 클릭 시 해당 지역 분석 또는 실거래 상세 URL로 이동한다. WebSocket/STOMP 확장은 다음 contract 후보로 둔다.

## 인사이트 지표 방향

기존 금융 화면의 지수/등락률 패턴을 그대로 가져오면 부동산에서는 의미가 흐려질 수 있습니다. 부동산은 거래 빈도가 낮고 신고·공개 지연이 있으며, 지역·단지·면적·연식에 따라 같은 기간의 가격 신호가 다르게 보입니다. 따라서 첫 화면은 지표 나열보다 "지금 확인할 시장 사실 묶음"으로 둡니다.

- 가격 흐름: 실거래가, 전세가, 공표 지수의 방향과 `asOf`, 표본 수, stale 여부를 함께 표시
- 거래 강도: 거래 건수, 신고 건수, 매물 소진 체감, 거래량 부족 caveat 표시
- 전세 압력: 전세가 변화, 전월세 비중, 전세 매물 감소/증가 후보
- 공급/청약: 입주 예정, 분양/청약 일정, 경쟁률, 미분양 후보 데이터
- 정책/교통 이벤트: 대출, 세제, 정비사업, 철도/도로 발표를 원인 확정이 아닌 context로 표시
- 보조 공개 반응: 기대/우려/중립, 언급량, 별칭 확산, source skew를 가격 사실과 분리 표시하되 핵심 지표로 승격하지 않음
- 핵심 지역 흐름: 마포구, 성수동, 동탄역권, 송도처럼 사용자가 보는 지역 단위의 월/3개월/6개월/년 변화율로 표시

카드 라벨은 `상승률`, `강세` 같은 결론형 표현보다 `실거래 공개`, `전세 압력`, `거래 강도`, `정책 이벤트`, `주요 일정`처럼 사용자가 직접 판단할 수 있는 이름을 우선합니다.

## 상태와 빈 화면

- loading: market summary skeleton, timeline skeleton, stale badge placeholder
- empty: 확인할 시장 fact 없음, provider 상태와 mock 여부 표시
- error: 수집/API 실패 영역별 표시
- stale/mock: provider, `asOf`, `stale`, `mock` badge를 카드마다 표시

## API 후보

| 필드 | 소유 도메인/layer | 설명 |
| --- | --- | --- |
| `targets[]` | realestate/backend | 확인할 지역/단지 후보 |
| `targets[].marketFacts` | realestate | 실거래/전세/매물/정책 fact와 provider/asOf/stale |
| `targets[].scheduleSummary` | realestate/backend | 공식 통계, 청약, 정책 일정 요약 |
