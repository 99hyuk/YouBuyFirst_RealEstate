# 대시보드 화면

## Route

- Parent: root
- Route 정보: `/dashboard`
- Child screens: `/realestate/map`, `/realestate/transactions`, `/realestate/targets/:targetId`, `/indicators`, `/newsroom`

## 화면 목적

첫 화면에서 오늘의 핵심 부동산 상황, 공식 데이터 상태, 뉴스/리포트/영상/공식 링크를 빠르게 훑고 상세 화면으로 이동합니다. 첫 블록은 특정 지역 상승률 차트가 아니라 실거래·전세·공급·정책 흐름과 최근 이슈를 짧게 엮은 일일 브리핑으로 둡니다. 주요 지표 나열은 `/indicators`에서 제거하고, `/indicators`는 공식 발표 일정 화면으로 사용합니다. 화면 문구는 매수·매도 지시가 아니라 공개 데이터와 근거를 함께 보는 관찰형 정보로 유지합니다. 단지/아파트 후보는 공식 좌표와 실거래가 연결된 경우에만 보조로 노출합니다.

## 현재 섹션

- 검색: 검색창 위에 `너나사 YouBuyFirst` 브랜드 타이틀을 크게 노출하고, 제목 아래 태그라인은 `당신을 위한 부동산 인사이트, 너나사`로 둬서 브랜드명은 유지하되 부동산 서비스임을 감성적으로 알림. 상단 영문 보조 라벨은 태그라인과 중복되므로 노출하지 않음. 지역 검색 진입점을 우선 두고, 단지 검색은 좌표/실거래/언급이 연결된 후보만 상세로 보냄. 검색창 주변에는 반응 필터, 집계 메타 칩, 기간·반영 상태 eyebrow, 특정 지역 쏠림을 단정하는 자동 헤드라인을 노출하지 않음
- Today 3줄 브리핑: AI가 최근 핵심 쟁점, 공식 지표, 뉴스/리포트 후보를 큰 문장 3개로 요약함. 제목의 `Today`는 브랜드 포인트 컬러로 표시하고, 제목과 `자세한 분석 보러가기` CTA는 요약 카드 밖에 두고 같은 제목선 높이로 정렬함. 큰 외곽 박스 없이 01, 02, 03 숫자 마커와 핵심 문장만 강하게 보여주며, 소제목/보조설명 투트랙 문장은 쓰지 않음. 개별 카드별 바로가기는 두지 않음. 카드 높이는 옆 패널에 억지로 맞추지 않고 문장 밀도에 맞춰 과한 여백이 생기지 않게 유지함
- 지역 인사이트 한눈에: 가격 흐름, 전세 압력, 거래 강도, 정책/공급 이벤트가 함께 관찰되는 지역 묶음. 단지 후보는 보조 리스트로만 사용. 기대/우려/중립 아이콘 범례처럼 카드 내용을 반복 설명하는 보조 UI는 두지 않음
- 뉴스/리포트/영상/링크 feed: 실제 source icon 기반 외부 링크. 카드 헤더에는 영어 kicker나 `콘텐츠 반영` 같은 반복 상태 pill을 두지 않고, 점 아이콘 옆에 한글 제목을 단일 행으로 표시하며 `자세히 보기` 링크만 둠. 영상과 블로그·커뮤니티 목록은 순위형이 아니라 일반 콘텐츠 목록으로 보여주며 `1위`, `2위` 같은 번호를 붙이지 않음
- 오른쪽 빠른 패널: `반응/지표/관심` 탭 전환 UI와 `라이징 스타` 보조 카드를 두지 않고, 현재 확인할 핵심 지역 흐름 카드 하나만 보여줍니다. 지표 상세는 `/indicators`, 실거래 탐색은 `/realestate/transactions`, 저장 지역 관리는 `/realestate/mypage`에서 처리합니다.
- 공통 footer: `커뮤니티 반응`, `지역·단지 언급량`, `에이전트 근거 로그` 중심 설명을 쓰지 않고, 실거래·전세 흐름, 지도 기반 지역 변화, 주요 일정, 뉴스·리포트 이슈를 함께 보는 부동산 인사이트 서비스로 설명합니다.

## 빠른 패널 거래/일정 기준

- 거래 탐색: 최근 매매/전월세 거래 수, 최신 거래일, 공식 거래 없음 상태
- 주요 일정: 가격지수, 실거래 공개, 미분양·공급, 금리, 청약, 정책 발표
- 지도 흐름: 공식 가격지수 또는 검증된 market fact 기반 상승/하락
- 보조 반응: 공개 원문에서 반복되는 쟁점 후보가 있을 때만 축약 표시

## 상태와 빈 화면

- loading: 일일 브리핑, 지도/거래 요약, 일정 카드, feed skeleton을 먼저 보여줍니다.
- empty: 관찰 지역이 없으면 지역 분석 화면과 마이페이지 저장 흐름으로 이동할 수 있게 안내합니다.
- error: 뉴스, 공공데이터, 공식 일정, 보조 원문 수집 실패를 영역별 badge로 분리합니다.
- stale/mock: 실거래·공공통계는 공개 지연 가능성을 항상 표시합니다.
