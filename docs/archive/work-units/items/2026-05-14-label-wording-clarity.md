# 작업 단위: 라벨 설명 명확화

## 목적

`type:*`, `track:*`, `part:*` 라벨이 서로 비슷하게 보이지 않도록 설명 단어를 고정합니다.

## 포함 범위

- `track:*`는 작업 트랙으로 설명합니다.
- `type:*`는 작업 타입으로 설명합니다.
- `part:*`는 변경 파트로 설명하고, 필수가 아닌 보조 라벨임을 명확히 합니다.
- PR 템플릿, Git/PR 컨벤션, 라벨/태그 사전, 워크플로 문서, Notion 태그 사전을 같은 표현으로 맞춥니다.
- GitHub 실제 라벨 설명도 같은 단어를 쓰도록 갱신합니다.

## 제외 범위

- 라벨 체계 자체의 추가 변경
- backend, pipeline, front 기능 구현
- Notion 데이터베이스 schema 변경

## 검증 기준

- 설명 문서에서 `작업 트랙`, `작업 타입`, `변경 파트` 표현을 확인할 수 있습니다.
- Notion 라벨/태그 사전도 같은 표현을 사용합니다.
- GitHub 라벨 설명도 같은 표현을 사용합니다.
- `git diff --check`가 통과합니다.
