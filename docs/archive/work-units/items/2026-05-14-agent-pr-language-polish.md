# 작업 단위: AGENTS 범위 설명과 PR 문장 체계 정리

## 목적

병렬 작업이 시작된 뒤에도 새 채팅이 `AGENTS.md`를 읽고 자기 역할을 바로 이해할 수 있도록 안내 문장을 정리합니다. PR 본문도 사람에게 덜 어색하게 읽히도록 섹션명을 다듬습니다.

## 포함 범위

- `AGENTS.md`의 프로젝트 범위를 “현재 MVP만 구현”이 아니라 “최종 제품을 트랙별로 단계 구현”으로 수정합니다.
- GitHub 라벨의 병렬 작업 표기를 `stream:*`에서 `track:*`로 바꿉니다.
- `part:*` 라벨은 필수 라벨이 아니라 리뷰 경로를 돕는 보조 라벨로 정리합니다.
- PR 템플릿의 `왜 이 단위인가` 섹션을 `PR 범위`로 바꾸고, 포함/제외 범위 중심으로 작성하게 합니다.
- Notion 허브와 라벨/태그 사전도 같은 용어로 맞춥니다.

## 제외 범위

- backend, pipeline, front 기능 구현
- Notion 데이터베이스 schema 변경
- 트랙 자체의 추가/삭제

## 검증 기준

- 새 작업자가 `AGENTS.md`만 읽어도 현재 실행 기반과 병렬 트랙 구조를 구분할 수 있습니다.
- GitHub/Notion/저장소 문서에서 병렬 작업 라벨은 `track:*`로 보입니다.
- PR 본문 템플릿이 더 자연스러운 섹션명과 문장 구조를 씁니다.
