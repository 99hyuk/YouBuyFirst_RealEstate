# 작업 단위: PR/Notion 카드형 기록 정리

## 목적

PR과 Notion 작업 기록이 다시 텍스트만 길게 이어지는 형태로 돌아가지 않도록, 제목/본문/라벨/태그 기준을 문서와 템플릿에 고정합니다.

## 포함 범위

- PR 제목을 `[트랙][타입] 명사형 요약`으로 통일합니다.
- PR 템플릿에 라벨/태그 사전 링크와 카드형 섹션을 명확히 둡니다.
- GitHub 라벨과 Notion 태그 의미를 `docs/LABEL_GUIDE.md`에 모읍니다.
- 에이전트 가이드와 워크플로 문서가 같은 기준을 가리키도록 맞춥니다.

## 제외 범위

- GitHub Actions 동작 변경
- backend, pipeline, front 기능 구현
- Notion 데이터베이스 schema 변경

## 검증 기준

- 변경 문서에서 라벨/태그 의미를 따라갈 수 있습니다.
- 새 작업자가 PR 제목 형식과 카드형 본문 구조를 바로 이해할 수 있습니다.
- `git diff --check`가 통과합니다.
