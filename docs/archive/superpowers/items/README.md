# Superpowers 기록 안내

이 폴더는 Superpowers workflow로 작성했던 프로젝트 설계와 실행 계획의 장기 기록입니다. 플러그인 자체 사용 설명서가 아니며, 새 채팅 시작 루틴에서 전문 읽지 않습니다.

## 역할

- `specs/`: 당시의 설계 판단, 데이터/API 계약, 책임 분리, 검증 기준을 남긴 기록입니다.
- `plans/`: 당시 설계를 실제 작업 단위로 쪼갠 실행 계획과 PR 준비 기록입니다.
- 현재 정본이 아니라 과거 근거 저장소입니다. 최신 상태나 다음 행동은 `docs/current/HANDOFF.md`와 `docs/current/TASKS.md`를 먼저 확인합니다.

## 쓸 때

- 예전 결정의 이유가 필요할 때: Notion/PR 운영 구조, 수집 정책, CrawlTarget, AI mention resolver 같은 설계 배경을 확인합니다.
- 중단된 기능을 되살릴 때: 당시의 acceptance criteria, 테스트 범위, API shape를 참고합니다.
- 포트폴리오/기술 경험 근거가 필요할 때: 어떤 문제를 왜 그렇게 풀었는지 확인합니다.
- 현재 문서와 코드가 충돌할 때: archive는 근거 후보일 뿐이며, 현재 코드와 current/domain 문서가 우선입니다.

## 쓰지 않을 때

- 새 채팅 시작 루틴으로 읽지 않습니다.
- 현재 API 명세, 화면 명세, 작업 지시로 그대로 사용하지 않습니다.
- 플러그인 사용법이나 Superpowers 스킬 설명서로 보지 않습니다.
- 이미 current/domain 문서에 승격된 내용이면 archive를 다시 읽지 않습니다.

## 읽는 법

- 필요한 키워드가 있을 때만 `rg`로 찾습니다.
- `plans/`와 `specs/` 파일은 큰 기록이므로 관련 섹션만 봅니다.
- 새 계획을 실행할 때만 해당 plan 파일을 필요한 범위로 읽습니다.
- 대화에는 전문을 붙이지 말고 결정, 계약, 검증 기준만 요약합니다.

## 정리 기준

다음 중 하나라도 있으면 보관합니다.

- 현재 문서로 아직 승격되지 않은 설계 근거가 있다.
- DB/API/상태값/테스트 기준처럼 다시 구현할 수 있는 계약 정보가 있다.
- PR, Notion, 포트폴리오 기록의 근거가 된다.

모두 current/domain 문서에 반영됐고 과거 이력 가치도 낮다면 요약 후 삭제 후보로 둡니다.

## 큰 파일 주의

일부 plan/spec 파일은 3k-6k 토큰 이상입니다. 과거 근거가 필요할 때만 열고, 대화에는 요약만 남깁니다.
