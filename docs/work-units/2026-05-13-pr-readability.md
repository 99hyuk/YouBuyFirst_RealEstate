# Work Unit: PR Readability

## Goal

앞으로 생성되는 PR이 명령어 나열보다 사람이 이해하기 쉬운 설명과 검증 결과 중심으로 작성되도록 템플릿과 컨벤션을 개선한다.

## Scope

- GitHub PR 템플릿을 `한눈에 보기`, `바뀐 내용`, `왜 이 단위인가`, `검증 결과`, `리스크와 후속 작업` 구조로 변경
- 검증 명령은 접힌 영역에 두고, 본문에는 통과 결과와 확인 사실을 먼저 쓰도록 안내
- Git/PR 컨벤션과 보수적 개발 워크플로 문서에 같은 원칙을 반영
- 현재 handoff와 작업 목록에 새 운영 기준을 기록

## Out of Scope

- CI 동작 변경
- PR 승인 자동화 구현
- 크롤러, backend, worker 기능 변경

## Verification

- 문서 변경만 포함하므로 `git diff --check`로 공백 오류를 확인한다.
- PR #5 본문을 같은 기준으로 다시 정리해 실제 예시로 삼는다.

## Notes

PR 설명은 리뷰어가 먼저 의도와 결과를 이해하고, 필요할 때 명령어를 펼쳐보는 흐름을 기본으로 한다.
