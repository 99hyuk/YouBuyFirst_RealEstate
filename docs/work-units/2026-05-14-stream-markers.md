# 작업 단위: 병렬 작업 트랙 표시 기준

## 목적

여러 채팅과 에이전트가 동시에 작업할 때 GitHub, PR, Notion에서 어느 트랙의 작업인지 바로 구분할 수 있게 표시 체계를 정합니다.

## 범위

- PR 템플릿에 `track:` 라벨 행 추가
- Git/PR 컨벤션에 트랙 라벨과 브랜치 prefix 정리
- 워크플로와 에이전트 가이드에 `track:*` 라벨 필수화
- 작업 트랙 문서에 Notion `트랙` 속성 기준 추가

## 제외

- 실제 기능 구현
- 트랙별 코드 분리
- 새로운 서비스나 패키지 생성

## 검증 기준

- `git diff --check`가 통과합니다.
- GitHub에 `track:*` 라벨이 존재합니다.
- Notion 작업 로그 DB와 다음 작업 DB에 `트랙` 속성이 존재합니다.
