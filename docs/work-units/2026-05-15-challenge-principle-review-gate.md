# 작업 단위: 반박 원칙과 문서 미준수 사고 게이트

## 목적

사용자가 요청한 "무조건 수용하지 말고 의문을 제기하고 반박하라"는 협업 방식을 최우선 원칙으로 올립니다. 또한 문서에 이미 있는 원칙을 따르지 않았을 때 왜 문제가 생겼는지 분석하고 재발 방지를 남기는 절차를 고정합니다.

## 범위

- `AGENTS.md`에 최우선 협업 원칙과 문서 미준수 사고 대응 절차 추가
- `docs/CHAT_START_GUIDE.md`에 반박과 토론이 필요한 경우와 응답 틀 추가
- `docs/WORKFLOW.md`에 문서 미준수 사고 분석 게이트와 Notion 정기 점검 책임 추가
- `docs/CURRENT_HANDOFF.md`에 다음 에이전트가 반드시 따라야 할 규칙으로 반영
- `docs/FINAL_PRODUCT_PLAN.md`의 데이터 도메인 용어를 `stock`, `analysis`, `indicator` 기준에 맞게 보정
- Notion 제품 기획과 작업 맥락, 채팅 시작 사용법 페이지에 같은 기준 반영

## 하지 않는 일

- 실제 코드 패키지 리네임
- Notion 루트 대시보드 구조 변경
- 새로운 자동화 생성

## 완료 기준

- 새 에이전트가 `AGENTS.md`와 `CURRENT_HANDOFF.md`만 읽어도 반박 원칙을 최우선으로 인지합니다.
- 문서 미준수 문제가 발생하면 원인, 손상 범위, 복구, 재발 방지를 확인해야 한다는 절차가 명시됩니다.
- Notion 제품 기획 페이지의 데이터 도메인명이 repo 기획 문서와 맞습니다.

## 검증

- `git diff --check` 통과
- 관련 문서에서 `최우선 협업 원칙`, `문서 미준수 사고 분석 게이트`, `AnalysisResult`, `IndicatorSnapshot` 검색 확인
- Notion 제품 기획과 작업 맥락, 채팅 시작 사용법 페이지 fetch 확인
