# 프론트/기획 우선 긴급 작업

작성일: 2026-06-22 KST

## 결론

지금 MVP는 크롤링과 지역 반응을 중심으로 더 확장하기보다, 사용자가 바로 이해할 수 있는 `실거래`, `지도 기반 지역 흐름`, `주요 일정`, `근거 브리핑` 중심으로 정리해야 한다.

부동산은 주식처럼 초단기 반응 등락을 보는 서비스가 아니다. 실거래, 전세, 공급, 정책, 청약 일정처럼 드러나는 시장 사실을 정확히 필터링하고, 그 위에 해석을 붙이는 쪽이 더 설득력 있다. 공개 커뮤니티 반응은 있으면 보조 관찰 신호로만 쓰고, 별도 핵심 탭이나 대표 랭킹으로 세우지 않는다.

## 바로잡아야 할 제품 기준

1. 지도와 실거래를 중심축으로 둔다.
   - 지도는 지역별 흐름과 drilldown을 담당한다.
   - 실거래는 지역, 기간, 거래 유형, 가격, 면적 기준의 필터링과 목록 확인을 담당한다.
   - 단지 단위는 공식 좌표와 공식 거래가 연결된 경우에만 보조 상세로 확장한다.

2. 주요 지표 페이지는 주요 일정 페이지로 전환한다.
   - 지표 나열은 사용자가 반복해서 볼 이유가 약하다.
   - `/indicators`는 공식 통계, 실거래 공개, 청약, 정책, 금리 일정과 출처 링크를 보여준다.
   - 핵심 지표 값은 대시보드나 상세 리포트 안에서 필요한 만큼만 요약한다.

3. 지역 반응 페이지는 실거래 페이지로 전환한다.
   - 신규 정본 route는 `/realestate/transactions`로 둔다.
   - 기존 `/realestate/reactions`는 전환 기간의 redirect 또는 호환 route로만 둔다.
   - 반응 랭킹은 핵심 화면이 아니라 상세 리포트와 근거 로그의 보조 정보로 내린다.

4. 데이터 부족을 숨기지 않는다.
   - 공식 거래 없음, 좌표 미검증, stale, provider 지연, 표본 부족은 화면에서 보여준다.
   - 빈 값을 mock으로 채우지 않는다.
   - 공식 데이터가 없으면 `공식 거래 확인 전`, `공식 데이터 없음`, `데이터 확인 전`으로 표현한다.

## P0 프론트 수정

### 대시보드

- 첫 화면은 `오늘 확인할 시장 fact`, `주요 일정`, `뉴스/리포트 근거`, `지도/실거래 진입`을 중심으로 정리한다.
- 반응 필터, 언급 합계, 단정형 헤드라인, 임시 gauge는 추가하지 않는다.
- 핵심 지표는 카드 나열보다 브리핑과 요약 패널 안에서 필요한 만큼만 보여준다.
- 뉴스, 리포트, 영상, 블로그 영역은 비어 있으면 mock 글을 채우지 말고 `수집 대기` 상태를 표시한다.

### 지도

- 지도 색상은 상승 빨강, 하락 파랑, 데이터 없음은 어두운 회색 계열로 통일한다.
- 데이터가 없는 시군구는 클릭 리포트를 열지 않거나 `공식 지표 없음` 리포트로 제한한다.
- 기간 버튼은 월/3개월/6개월 중심으로 재정리한다.
- 지도는 지역 분석의 주 화면이므로, 지역 반응 화면이 하던 지역 비교 역할을 흡수한다.

### 실거래

- `/realestate/transactions` 화면을 새 정본으로 둔다.
- 필터는 지역, 기간, 거래 유형, 가격대, 면적, 데이터 상태를 우선한다.
- 결과 리스트는 거래일, 지역/단지, 유형, 면적, 층, 가격, provider, asOf, stale 상태를 표시한다.
- 요약 카드는 거래 수, 중위 거래가, 직전 기간 대비 변화, 전세 압력, 공개 지연 상태를 보여준다.
- row 클릭은 `/realestate/targets/:targetId` 또는 지도 상세로 연결한다.

### 주요 일정

- `/indicators`는 지표 상세가 아니라 일정 캘린더로 운영한다.
- 가격지수·거래현황, 공급/미분양, 금리, 청약, 정책·공시가격 발표 일정을 provider/source URL과 함께 표시하고, 실거래 공개는 freshness/status로 분리한다.
- 기존 `/indicators/:category`는 북마크 호환 route로 같은 일정 화면을 보여준다.

### 단지/지역 상세

- 상세 화면은 시장 fact, 일정, 뉴스/리포트 근거, AI 근거 로그를 중심으로 구성한다.
- 단지 위치 레이어 아래의 `최근 매매 거래`는 공식 데이터만 표시한다.
- 공식 거래가 없으면 빈 상태를 보여주고, SSAFY HOME 과거 거래를 최신 거래처럼 보여주지 않는다.
- 공개 반응은 보조 섹션으로만 두고, 상세 결론의 핵심 근거처럼 쓰지 않는다.

### 뉴스룸

- 뉴스, 리포트, 영상, 블로그/공개 원문을 같은 feed로 뭉개지 않는다.
- 정책/통계/리포트는 한국부동산원, 국토교통부, KB/은행/증권사/연구기관 자료 중심으로 분리한다.
- 유튜브는 부동산 분석, 정책 토론, 시장 해설 채널 중심으로 선별한다.
- 최신 글이 없으면 오래된 글을 최신처럼 보이지 않게 날짜와 상태를 강조한다.

## P0 기획 수정

1. 서비스 핵심 문장을 인사이트 중심으로 정리한다.
   - `실거래, 전세, 매물, 공급, 정책 이슈를 지역 기준으로 엮어 보여주는 AI 부동산 인사이트 탐색 서비스`
   - MVP에서는 단지 전체 커버리지보다 지역/생활권 중심이라는 제한을 명확히 둔다.

2. 화면 정의서에서 탭 역할을 바꾼다.
   - `지역 반응` -> `실거래`
   - `주요 지표` -> `주요 일정`
   - `지역 분석`은 지도 탭이 담당한다.

3. 데이터 신뢰도 상태를 추가한다.
   - 공식 데이터 있음
   - 공식 데이터 없음
   - 공개 지연
   - 표본 부족
   - 좌표 미검증
   - 검색 후보
   - AI 평가 대기

4. 보조 공개 원문 기준을 낮은 우선순위로 둔다.
   - 토론/질문/후기형 원문은 보조 반응 신호로만 사용한다.
   - 실거래 표/신고가 목록형 원문은 반응 지표가 아니라 시장 fact 후보 또는 제외 대상으로 관리한다.
   - 검색 API와 SerpApi 결과 수는 관심도 점수가 아니라 근거 링크 후보로만 사용한다.

## P1 데이터 작업

- 지도와 실거래에 필요한 지역부터 공공데이터 backfill 우선순위를 정한다.
- 대시보드 후보 지역, 지도 drilldown 지역, 실거래 기본 지역 순서로 공식 거래/전월세/가격지수/미분양 데이터를 채운다.
- 공공 API가 0건을 반환한 지역은 0건 자체를 기록하고, 화면에는 데이터 없음으로 표시한다.
- 원문 수집 결과에는 `토론`, `실거래 표`, `광고/분양`, `뉴스/리포트`, `검색 후보` 같은 유형 분류를 남긴다.

## 완료 기준

- 사용자가 화면만 봐도 어떤 값이 실제 데이터이고 어떤 값이 부족한지 알 수 있다.
- 지도, 실거래, 상세 화면이 같은 target 기준을 사용한다.
- 지역 반응 탭은 신규 제품 정본에서 빠지고, `/realestate/transactions`가 탐색 화면 역할을 한다.
- 주요 지표 나열 페이지는 빠지고, `/indicators`가 주요 일정 화면으로 동작한다.
- mock 값은 남아 있더라도 명확히 빈 상태, 수집 대기, 확인 필요로 표현된다.
