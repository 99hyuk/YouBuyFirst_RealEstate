Definition of Done

  • 대시보드에서 시장 fact와 주요 일정이 API 또는 명확한 mock fallback으로 보인다.
  • 실거래 탐색에서 지역, 기간, 가격, 면적 조건으로 거래 row를 확인할 수 있다.
  • 지역/단지 상세에서 provider, asOf, stale, 근거 링크, caveat가 함께 보인다.
  • 백엔드 테스트, 프론트 테스트/build, 데이터 bootstrap smoke를 수행한다.
  • 발표자는 데이터 지연, 공식 데이터 없음, 보조 반응의 한계를 설명할 수 있다.

주요 위험과 대응

  • 공공데이터 지연: 실시간처럼 표현하지 않고 기준시각과 stale 상태를 분리한다.
  • 데이터 중복: provider key, alias, target graph 기준으로 중복 merge 후보를 관리한다.
  • 시연 실패: mock fallback과 sample seed를 준비하고 API smoke URL을 별도 보관한다.
  • AI 과장 표현: 행동 권유, 상승 단정, 수익 보장 문구를 guardrail 테스트로 차단한다.
  • 수집 정책 리스크: 공개 HTTP 우선, 로그인/CAPTCHA/프록시 우회 금지를 문서와 코드 기준으로 유지한다.
