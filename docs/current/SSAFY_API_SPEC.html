SSAFY 관통프로젝트 제출 문서

너나사 부동산 API 명세서

실거래, 전세, 공급, 정책, 뉴스/리포트 이슈를 지역과 생활권 기준으로 연결해 보여주는 부동산 인사이트 서비스의 백엔드 API 명세입니다. 문서는 현재 Spring Boot Controller와 DTO 기준으로 작성했습니다.

프로젝트YouBuyFirst RealEstate
작성 기준일2026-06-25
BackendSpring Boot 3.3, Java 21
FrontendVue 3, Vite, TypeScript

1. 문서 범위

이 명세서는 화면에서 직접 쓰는 공개 API와 데이터 적재/운영 확인을 위한 internal API를 분리해 정리합니다. 투자 권유나 가격 상승 단정이 아니라, 공식 데이터와 근거 로그를 사용자에게 보여주는 관찰형 서비스 계약을 기준으로 합니다.

구분 포함 내용 대표 endpoint
인증/세션 회원가입, 로그인, 현재 사용자, 로그아웃, OAuth provider 상태 /api/auth/*
부동산 공개 API 지역/단지 검색, 시장 사실, 지도 layer, 리포트, 근거 로그, 일정, 콘텐츠 /api/realestate/*
채팅 API 게스트/회원 채팅 메시지, 접속 수, 닉네임 확인, SSE 스트림 /api/chat/*
Internal API 공공데이터 적재, 수집 결과 저장, batch 실행, alias/graph/materialized data 갱신 /internal/*, /admin/*
Swagger 확인 경로

Springdoc 의존성이 포함되어 있어 로컬 서버 실행 후 http://localhost:8080/swagger-ui.html에서 자동 생성 명세를 확인할 수 있습니다. OpenAPI JSON 기본 경로는 /v3/api-docs입니다.

201 Created새 리소스 생성회원가입, 새 관심 대상 저장, 채팅 메시지 생성
202 Accepted비동기 작업 접수좌표 사전 적재 trigger
204 No Content응답 본문 없음로그아웃, 관심 대상 삭제, 최신 브리핑 없음
400 Bad Request요청값 검증 실패windowMinutes 범위 오류, Bean Validation 실패

409 Conflict중복/충돌회원가입 중 username, email, displayName 중복
Internal API 운영 주의

현재 SecurityConfig에서 인증 필수로 묶은 endpoint는 /api/auth/me, /api/auth/logout, /api/realestate/watch-targets 계열입니다. /internal/*와 /admin/*는 로컬 개발/운영자 네트워크 전용 API로 취급해야 합니다.

4. 인증 API

POST /api/auth/register

새 사용자를 만들고 바로 세션을 시작합니다. 성공 시 JSESSIONID 쿠키가 내려갑니다.

Request bodyusername, email, password, displayName
Validationusername: 영문/숫자 4-20자, password: 8-100자이며 영문/숫자/특수문자 포함, displayName: 2-20자
Response201 Created, CurrentUserResponse
Error400 validation 실패, 409 username/email/displayName 중복
 
{
  "username": "ssafy2026",
  "email": "ssafy@example.com",
  "password": "Passw0rd!",
  "displayName": "싸피유저"
}
POST /api/auth/login
Request bodyusername, password
Response200 OK, CurrentUserResponse
Error401 Unauthorized when credential mismatch
GET /api/auth/me Session required

현재 세션 사용자를 반환하고 마지막 접속 시각을 갱신합니다.

{
  "id": "user-01",
  "username": "ssafy2026",
  "email": "ssafy@example.com",
  "displayName": "싸피유저",
  "authProvider": "local",
  "createdAt": "2026-06-25T09:00:00Z",
  "lastSeenAt": "2026-06-25T09:10:00Z"
}
GET /api/auth/oauth/providers

OAuth provider별 설정 상태를 반환합니다. 실제 OAuth 진입은 응답의 authorizationUrl을 사용합니다.

Response itemprovider, displayName, authorizationUrl, configured
Related routeSpring Security OAuth2 기본 route: /oauth2/authorization/{provider}, callback: /login/oauth2/code/{provider}
POST /api/auth/logout Session required

SecurityContext를 지우고 서버 세션과 JSESSIONID 쿠키를 만료합니다. 응답은 204 No Content입니다.

공통 데이터 상태 필드

필드 의미 표현 원칙
provider 데이터 제공자. 예: MOLIT, REB, KB, HF 등 출처가 확인되지 않으면 임의로 채우지 않음
providerDataset provider 내부 dataset 구분. 예: apt_trade, apt_rent 수집/검증/재처리에 쓰이는 key
asOf 사용자에게 보여줄 기준 시점 실시간처럼 보이지 않게 화면에 함께 표시
stale 정해진 freshness 기준보다 오래된 데이터 여부 true이면 요약/판단에서 제한적으로 사용
dataStatus ok, partial, unknown, mock 등 데이터 상태 확인되지 않은 값은 추측하지 않고 상태로 분리
confidence alias, graph, 반응 snapshot, 평가 근거의 신뢰도 결론 강도가 아니라 근거 품질 보조값

8. 대표 사용자 흐름

흐름 호출 순서 화면 효과
대시보드 진입 /api/auth/me optional 확인 후 /api/realestate/daily-briefings/latest, /api/realestate/map/layers, /api/realestate/dashboard/market-summary, /api/realestate/newsroom 오늘 브리핑, 지도 요약, 시장 요약, 뉴스/리포트 feed를 한 화면에 구성
지역 상세 확인 /targets/{targetId}/regional-report, /evidence-logs, /content, /timeline, /nearby-complexes 지역 분석 문장과 그 문장의 출처, 뉴스/정책/단지 근거를 함께 표시
실거래 탐색 /api/realestate/market-facts, /api/realestate/targets/{targetId}/market-facts, 필요 시 /geocode 거래 유형, 지역, 기간, 면적/가격 조건에 맞는 공식 시장 사실과 좌표를 표시
관심 지역 저장 /api/auth/login 이후 POST /api/realestate/watch-targets, 목록은 GET /api/realestate/watch-targets 로그인 사용자의 관심 지역/단지 목록을 저장하고 topbar/화면에서 재사용
운영 데이터 갱신 Python pipeline이 /internal/realestate/public-data/raw-ingestions 적재 후 /promote-staging, 필요 시 /map/layer-snapshots/refresh 공식 데이터가 market fact와 지도 layer에 반영되고 SSE로 업데이트 알림 가능
