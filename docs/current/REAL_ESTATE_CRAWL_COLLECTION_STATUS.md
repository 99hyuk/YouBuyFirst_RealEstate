# 부동산 커뮤니티 크롤링 수집 상태

기준 시각: 2026-06-17 KST

## 쉬운 결론

- 300건은 서비스에 쓰기에는 너무 적다. 그래서 네이버 카페 수집은 10개 query basket, basket당 최대 200건으로 올려 1회 실행 기준 2,000개 후보를 확인하도록 바꿨다.
- 최종 네이버 카페 백필 결과는 후보 2,000건 중 1,429건 적재, 필터 제외 311건, 중복 링크 260건이다.
- 네이버 카페 1,429건 중 1,294개 게시글에서 지역/단지 mention이 잡혔고, reaction observation 15,815건, 7일 snapshot 1,572개(region 198, complex 1,374)가 생성됐다.
- 현재 전체 `community_posts`는 9,671건이고, 주요 source는 DCInside 5,407건, PPOMPPU 2,406건, NAVER_CAFE 1,429건, DAUM_CAFE 249건이다.
- MVP 시연에서 “어디가 많이 언급되는지”를 보여줄 정도의 신호는 쌓이기 시작했다. 다만 네이버 카페 신규 snapshot은 `source_skewed`가 대부분이라 운영 품질의 확정 지표로 보기에는 아직 부족하다.

## 네이버 카페 수집 보강

카페는 로그인/비공개 게시판을 긁지 않고 공식 공개 검색 API 결과만 discovery한다.

| 항목 | 값 |
| --- | ---: |
| query basket | 10 |
| basket당 요청 후보 | 200 |
| 전체 후보 | 2,000 |
| 적재 게시글 | 1,429 |
| 필터 제외 | 311 |
| 제목/본문 제외 | 295 |
| 키워드 미일치 | 16 |
| 중복 링크 | 260 |

query basket은 서울 핵심권, 경기 남부/북부, 인천, 지방 광역시, 전세/공급/미분양, 재건축/재개발, 청약/분양, 브랜드 단지, 대출규제/정책 축으로 나눴다.

광고성 품질 필터는 상담/문의형 문구, 대부/캐피탈/대환대출/무직자대출/주택담보대출 상품성 문구, 로열층 클린 매물 같은 매물 홍보성 문구를 제외하도록 강화했다. 최종 적재분 기준 `대출`, `상담`, `문의` 단순 키워드 잔여량은 0건으로 확인했다.

주의할 점은 Naver CafeArticle API 응답에는 게시글 작성일이 없다. 그래서 네이버 카페 게시글의 `publishedAt`은 실제 작성일이 아니라 수집 관측 시각 fallback이다. “최근 7일” 해석도 네이버 카페에 대해서는 실제 작성일 기준이 아니라 수집 기준으로 봐야 한다.

## reaction snapshot 반영

네이버 카페 최종 백필 입력으로 승인 alias 83,053개를 적용했다.

| 항목 | 값 |
| --- | ---: |
| 입력 게시글 | 1,429 |
| mention 매칭 게시글 | 1,294 |
| reaction observation | 15,815 |
| 7일 snapshot 전체 | 1,572 |
| region snapshot | 198 |
| complex snapshot | 1,374 |

리빌드 후 live API는 최신 네이버 7일 window를 보고 있다.

| type | latest window | snapshot |
| --- | --- | ---: |
| region | 2026-06-10 10:01:21 ~ 2026-06-17 10:01:21 | 198 |
| complex | 2026-06-10 10:01:21 ~ 2026-06-17 10:01:21 | 1,373 |

지역 ranking 상위 예시는 `서울특별시 강남구`, `서울특별시 서초구`, `경기도 화성시`, `경기도 광주시`, `경기도 의정부시` 순으로 잡힌다. API 응답에는 `sourceCount=1`, `sourceSkew=1.0`, `coverageStatus=source_skewed`가 함께 내려가므로 화면에서는 “언급은 많지만 단일 source 쏠림”을 같이 보여줄 수 있다.

단지 ranking은 브랜드 단독명이 여러 실제 단지로 번지는 문제가 있어 `힐스테이트`, `푸르지오`, `아이파크`, `센트럴파크`, `한양수자인` 같은 generic standalone complex name을 matcher와 API ranking 응답에서 제외하도록 보정했다. 같은 표시명이 여러 target으로 반복되는 경우도 ranking에서는 한 번만 보여준다. 최종 live API 기준 complex Top10은 `화성동탄2센트럴힐즈동탄아파트`, `센트럴푸르지오`, `금호어울림`, `한라비발디`, `파크푸르지오` 등으로 채워지며, 모두 `source_skewed` 상태다.

## 품질 판단

지금 표본은 MVP 시연에서 지역 반응 ranking, 쟁점 비율, source skew 표시를 보여주기에는 유용하다. 특히 300건 시점보다 네이버 카페 신호가 훨씬 두꺼워졌다.

다만 아직 운영 수준의 신뢰도는 아니다.

- source 쏠림이 크다. DCInside, PPOMPPU, NAVER_CAFE 비중이 대부분이다.
- 네이버 카페는 실제 작성일이 없어 시간축 정확도가 약하다.
- 단지 ranking의 대표적인 브랜드 단독명 과매칭은 화면 응답에서 막았지만, 아직 source skew가 크고 동/주소 context 없이 같은 공식명이 반복되는 단지는 추가 merge/정본화가 필요하다.
- 반복 수집이 아직 충분히 쌓이지 않아 “증가율”은 초기값 100%처럼 보일 수 있다.

다음 작업은 수집량 자체를 더 늘리는 것보다, 1일 1회 이상 반복 수집과 source 다변화, 단지 alias 정밀화, 카페/게시판별 품질 필터 리포트를 붙이는 쪽이 우선이다.
