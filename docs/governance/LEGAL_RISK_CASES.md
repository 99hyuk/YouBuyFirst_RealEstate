# 법적/운영 리스크 사례

이 문서는 부동산 전용 프로젝트에서 수집, 저장, 화면 노출, AI 평가가 건드릴 수 있는 법적/운영 리스크를 정리합니다. 법률 자문이 아니라 개발 중 지켜야 할 실무 기준입니다.

## 핵심 원칙

- 공개 HTTP 수집을 우선합니다.
- 로그인 세션 크롤링, CAPTCHA 우회, 프록시 회전, fingerprint 위장은 하지 않습니다.
- 원문 전문을 재게시하지 않습니다.
- 작성자 개인 식별이나 작성자별 성향 추적을 하지 않습니다.
- 특정 매수, 매도, 청약, 대출 행동을 권유하지 않습니다.
- 실거래/전세/매물/정책 데이터는 provider, `asOf`, `stale`, 지연 여부를 함께 표시합니다.

## 원문 저장 리스크

커뮤니티와 뉴스/컬럼 원문은 저작권과 약관 리스크가 있습니다.

허용 후보:

- 제목
- 일부 snippet
- URL
- 작성 시각
- 작성자 해시
- 원문 해시
- source/board metadata

피해야 할 것:

- 본문 전문 저장
- 원문을 대체할 만큼 긴 요약
- 닉네임/프로필/작성자별 추적
- 원문 이미지를 그대로 재게시

## 커뮤니티 수집 리스크

소스별로 robots, 약관, 접근 제한, 차단 정책이 다릅니다. 새 source는 기본 `disabled`로 두고 검토 후 상태를 올립니다.

네이버 카페와 다음 카페는 부동산 반응 source 후보가 될 수 있지만, 비로그인 공개 목록과 공개 글만 검토합니다. 로그인이 필요하거나 멤버십 가입, CAPTCHA, 앱 전용 접근, 우회성 헤더/fingerprint가 필요한 source는 구현하지 않습니다.

source 상태:

- `enabled`: 운영 정책 검토 후 공개 환경에서 수집 가능
- `public-demo-only`: 공개 화면에는 fixture, 지연 데이터, 샘플 데이터만 사용
- `local-research-only`: 로컬 연구와 시연용으로만 실행
- `disabled`: 약관, 접근 제한, 안정성 문제로 비활성화

## 부동산 데이터 노출 리스크

실거래/전세/매물/정책 데이터는 원천과 갱신 주기가 다릅니다. 지연된 데이터를 실시간처럼 보여주면 사용자 오해가 생깁니다.

필수 표시:

- provider
- `asOf`
- `stale`
- dataStatus
- 원천 링크 또는 출처
- 값 없음/확인 필요/mock 구분

## 행동 지시 표현 리스크

부동산 화면은 관찰형 분석 서비스로 보여야 합니다.

피해야 할 표현:

- 추천
- 사라
- 팔아라
- 수익 보장
- 진입
- 시그널 확정
- 오를 지역
- 무조건 오른다
- 청약 넣어라
- 대출 받아라
- 지금 사야 한다

허용 가능한 표현:

- 관심 급등
- 우려 증가
- 쟁점 비율
- 표본 신뢰도
- 유사 과거 상황
- 확인 필요
- 데이터 지연
- 출처 기준

## AI 평가 리스크

AI는 내부 추론 전문을 저장하거나 노출하지 않습니다. 사용자에게는 짧은 평가, 근거, caveat, 데이터 상태만 보여줍니다.

AI 출력에는 아래를 포함합니다.

- 근거 데이터
- provider/asOf/stale
- 표본 부족 또는 source skew caveat
- 관찰형 분석이라는 고지

## 업데이트 규칙

- 새 source를 추가하기 전에는 이 문서나 별도 source policy에 robots/약관/접근성/저장 범위를 남깁니다.
- 화면 문구가 행동 지시처럼 보이면 제품 문구 기준으로 되돌립니다.
- 법률 검토가 필요한 항목은 확정 구현으로 넣지 않고 `확인 필요` 상태로 둡니다.
