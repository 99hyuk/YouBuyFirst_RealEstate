# 커뮤니티 반응 용어 가이드

이 문서는 너나사 (YouBuyFirst)에서 커뮤니티 글을 분석할 때 쓰는 제품 용어와 수집 전략을 정리합니다.

## 용어 기준

너나사는 `감성`이라는 표현을 대표 용어로 쓰지 않습니다. 서비스가 보여주려는 것은 막연한 기분 분석이 아니라, 커뮤니티 글에서 관찰되는 투자 관련 반응을 데이터로 정리한 결과입니다.

대표 용어:

- `커뮤니티 반응`: 사용자 화면에서 쓰는 대표 표현
- `커뮤니티 반응 데이터`: 문서와 기술 설명에서 쓰는 상위 개념
- `반응 지표`: 화면과 API 설명에서 쓰는 집계 지표
- `반응 모멘텀`: 언급량과 반응 강도의 증가 흐름
- `반응 방향`: 단일 게시글 안에서 특정 종목이 어떤 방향으로 언급되는지 나타내는 값
- `개미 심리 지수`: 여러 커뮤니티 반응을 종목/기간 단위로 묶어 0-100 점수로 압축한 대표 지표
- `종합 커뮤니티 반응`: 종목/window 단위로 사용자에게 먼저 보여주는 하나의 결론
- `주요 쟁점`: 종합 커뮤니티 반응을 만든 이유나 말거리의 비율

사용자 화면에서는 `커뮤니티 반응`을 우선합니다. `커뮤니티 반응 신호`는 번역체처럼 들리고, 투자 서비스에서는 매수/매도 신호처럼 오해될 수 있으므로 대표 화면 문구로 쓰지 않습니다. 다만 코드와 문서에서 이벤트성 분석 결과를 설명할 때는 `신호`라는 단어가 제한적으로 남을 수 있습니다.

`개미 심리 지수`는 매수/매도 신호가 아니라 커뮤니티 반응의 온도계입니다. 언급량 변화, 낙관/비관 비율, 표현 강도, 인기글 확산, 소스 다양성, 표본 수 신뢰도, 시세 지연 상태를 함께 보고, 점수만 단독으로 노출하지 않습니다. 단순 낙관/비관 count는 baseline signal이며, 궁극적으로는 임베딩/클러스터링으로 묶은 `주요 쟁점` 비율과 대표 근거가 지표 신뢰도를 보강합니다.

종목 상세나 랭킹 화면에서는 `종합 커뮤니티 반응`을 하나의 결론으로 먼저 보여줍니다. 예를 들어 `낙관 우세`, `비관 우세`, `혼조`, `관망`처럼 표시하고, 아래에는 `AI 수혜 기대 38%`, `단기 고점 부담 19%`처럼 주요 쟁점의 비율을 붙입니다. 쟁점마다 낙관/비관 결론을 큰 문구로 반복하면 전체 판단이 흐려지므로, 쟁점 방향성은 색상이나 작은 보조 배지, 내부 산식용으로 제한합니다.

반응 방향 표시값:

| 내부 값 | 사용자 표시 | 의미 |
| --- | --- | --- |
| `bullish` | 낙관 | 상승 기대, 호재 기대, 긍정적 전망에 가까운 반응 |
| `bearish` | 비관 | 하락 우려, 악재 우려, 부정적 전망에 가까운 반응 |
| `neutral` | 중립 | 명확한 방향이 없거나 정보 공유에 가까운 반응 |

향후 코드/DB 이름을 정리할 때는 `stance`보다 `reactionDirection`을 우선 후보로 둡니다. `stance`는 분석가 관점의 단어라 짧지만, 새 작업자가 바로 이해하기에는 `reactionDirection`이 더 직관적입니다.

## 피해야 할 표현

- `감성 분석`: 필요하면 legacy 구현이나 과거 문서 설명에서만 사용합니다.
- `긍정/부정`: 투자 맥락에서는 감정 평가처럼 보이므로 `낙관/비관`을 우선합니다.
- `매수/매도 신호`: 투자 자문처럼 보일 수 있으므로 사용자 화면에서는 쓰지 않습니다. 단, `매수 언급`, `매도 언급`, `순매수`, `매수 잔량`, `매도 잔량`, `매수/매도 호가`처럼 출처가 있는 데이터 라벨은 사용할 수 있습니다.
- `추천`, `수익 보장`, `따라 사기`: 제품 목적과 법적 리스크 기준에 맞지 않습니다.

허용되는 표현의 기준은 단어 자체가 아니라 말하는 주체와 기능입니다. 금융 사이트처럼 체결, 호가, 개인/기관/외국인 순매수, 커뮤니티 원문 속 매수/매도 언급 흐름을 보여주는 것은 관찰 데이터입니다. 반대로 서비스가 `사야 한다`, `팔아야 한다`, `이걸 매수하는 건 바보다`처럼 사용자의 행동을 평가하거나 유도하면 제품 결론으로 읽힐 수 있어 피합니다.

짠맛/팩트폭격 한줄평은 제품 톤으로 사용할 수 있습니다. 단, 웃긴 포인트는 사용자의 매수/매도 행동을 조롱하는 쪽이 아니라 커뮤니티 반응, 지표, 이후 성과의 모순을 찌르는 쪽으로 둡니다.

예시:

- `커뮤니티 온도는 90도, 7일 수익률은 냉장고`
- `기대감은 상한가, 근거는 아직 개장 전`
- `말은 달리고, 차트는 산책 중`
- `매수 언급은 늘었지만 단기 성과는 아직 못 따라옴`
- `뜨겁긴 한데, 표본 수가 아직 컵라면 물 수준`

짠맛 카피에는 가능하면 기준 기간, 표본 수, 출처 수, 비교 지표를 함께 붙입니다. 과격한 커뮤니티 원문 표현은 서비스 판단처럼 쓰지 않고 `커뮤니티 원문 톤` 또는 인용으로 분리합니다.

종목 상세 화면의 시황/기술지표/재무 기반 팩트폭격 헤드라인은 커뮤니티 반응 요약과 다른 기능입니다. 해당 기준은 `docs/domains/agent/STOCK_DETAIL_HEADLINE.md`를 봅니다.

## 소스 구조별 수집 전략

모든 커뮤니티를 같은 방식으로 수집하지 않습니다. 소스가 시장 전체 이야기를 섞어 보여주는지, 종목별 방으로 나뉘는지에 따라 역할을 나눕니다.

### 소스 활성화 상태

커뮤니티 구조와 별개로, 각 소스는 공개 운영 리스크와 실행 환경에 따라 활성화 상태를 가집니다. 현재 source policy와 `CrawlTarget` 실행 기준은 `COLLECTION.md`를 봅니다.

| 상태 | 수집 실행 | 공개 화면 | 기준 |
| --- | --- | --- | --- |
| `enabled` | 허용 | 집계 지표 표시 가능 | 운영 정책 검토가 끝난 소스 |
| `public-demo-only` | 공개 환경에서는 외부 요청 금지 | fixture/sample 표시 가능 | 공개 시연용 대체 데이터 |
| `local-research-only` | 로컬/비공개 연구에서만 허용 | 공개 노출 금지 | 검토 전 실험 소스 |
| `disabled` | 금지 | 숨김 또는 비활성 표시 | 위험, 차단, 접근 불가, 미검토 소스 |

새 소스는 기본적으로 `disabled`에서 시작합니다. 현재 MVP 소스도 공개 배포 전에는 별도 검토 없이 `enabled`로 올리지 않습니다.

### 일반 게시판형

대상 예시:

- 에펨코리아 주식 게시판
- 뽐뿌 증권포럼
- 디시인사이드 미국 주식 갤러리
- 디시인사이드 주식갤러리/국내주식 계열 갤러리

source registry 후보:

| 우선순위 | 후보 | 쓰임 | 주의점 |
| --- | --- | --- | --- |
| 1차 | 에펨코리아 주식 게시판 | 일반 글 흐름, 조회/댓글 기반 확산 확인 | 광고성 글과 밈성 표현 노이즈 |
| 1차 | 디시인사이드 미국 주식 갤러리 | 미국 주식 언급, 은어/밈 반응 | 개념글과 최신 글을 분리해야 함 |
| 1차 | 디시인사이드 주식갤러리/국내주식 계열 | 국내 주식 반응, 단기 이슈 포착 | 갤러리명/게시판 구조 변경 가능성 |
| 1차 | 뽐뿌 증권포럼 | 국내 투자자 일반 반응, 추천/조회 확산 | 게시판 정책과 추천 수 의미 확인 필요 |
| 2차 | 클리앙 주식한당 | 장문 토론, 보수적 투자 관점 보강 | 공개 접근성, 글 빈도 확인 필요 |
| 2차 | 팍스넷 커뮤니티/전체글/강추이종목 | 오래된 주식 커뮤니티 반응, 추천 게시글 | 홍보성/작전성 노이즈 필터 필요 |
| 2차 | 딴지 주식클럽 | 커뮤니티성 강한 일반 반응 | 공개 접근성, 게시글 빈도 확인 필요 |
| 2차 | 네이버 카페형 미국주식 커뮤니티 | 대형 미국주식 커뮤니티 후보 | 로그인/가입 장벽이 있으면 수집 제외 |
| 3차 | 오늘의유머 경제, 개드립 주식/재테크, 이토랜드 주식, MLB파크 불펜 내 주식 글 | 대형 커뮤니티 안의 산발적 주식 반응 | 종목 인식 난이도와 노이즈가 큼 |

위 후보는 객관적 MAU 순위가 아니라 전체 사이트 규모, 주식 게시판 특화성, 게시글 빈도, 공개 접근성, 수집 정책 리스크를 함께 본 실행 우선순위입니다. 새 후보는 바로 adapter 구현 대상으로 보지 않고, 기본 `disabled` 상태에서 robots/약관, 목록 공개 여부, HTML 안정성, 최신 글 수, 조회/추천/댓글 필드 존재 여부를 먼저 확인합니다.

역할:

- 시장 전체 반응 흐름 파악
- 새로 언급이 튀는 종목 발견
- 라이징 스타 후보와 테마 후보 생성

수집 방식:

- 최근 글 목록을 30분 단위로 수집합니다.
- 제목과 제한 snippet에서 종목명, 티커, 별칭을 인식합니다.
- 언급량 증가율, 키워드 변화, 반응 방향 분포를 이용해 후보 종목을 만듭니다.
- 개미 심리 지수는 종목/기간/window별로 계산하고, 표본 수가 낮거나 소스 편중이 큰 경우 점수를 숨기거나 주의 상태로 표시합니다.
- 인기글, 개념글, 추천글, 조회수 상위글은 일반 최신 글 수집을 대체하지 않고 확산 확인 레이어로 수집합니다.
- 조회수, 추천수, 댓글수는 소스마다 의미가 다르므로 raw 수치끼리 직접 비교하지 않고 소스 내부 percentile, 시간 대비 증가율, 종목 관련도를 함께 봅니다.
- 디시인사이드 계열은 갤러리 이름이 비슷하거나 바뀔 수 있으므로 코드에 이름을 하드코딩하지 않고 `CrawlTarget` 또는 source registry에서 `source`, `boardId`, `displayName`, `marketScope`, `crawlPolicy`로 관리합니다.

### 종목 게시판형

대상 예시:

- 네이버 종토방
- 토스 종목 커뮤니티

역할:

- 특정 종목에 대한 깊은 반응 확인
- 일반 게시판형에서 발견한 후보 종목 검증
- 종목별 보유자 불안, 기대, 이슈 반응 보강

수집 방식:

- 모든 종목을 30분마다 전수 수집하지 않습니다.
- `CrawlTarget` 큐로 우선순위를 관리합니다.
- 우선순위는 일반 게시판 언급 증가, 거래대금, 급등락, 사용자 관심 종목, 최근 활동량, 실패 backoff를 함께 반영합니다.

## 반응 데이터 사용 방식

| 목적 | 우선 소스 | 보조 소스 |
| --- | --- | --- |
| 시장 전체 반응 | 일반 게시판형 | 시세/거래량 |
| 라이징 스타 후보 | 일반 게시판형 | 종목 게시판형, 시세/거래량 |
| 종목별 반응 상세 | 종목 게시판형 + 일반 게시판형 | AI 재서술 근거 |
| 이슈 확산 확인 | 인기글/개념글/추천글 | 유튜브, 신뢰 블로그, 뉴스/공시 링크 |
| 통합 지표 기반 성과 검증 | 종합 커뮤니티 반응, 내부 source-slice 반응 지표 | 이후 가격 변화 |

이 기준은 “수집 가능한 모든 글을 긁는 서비스”가 아니라, 투자 참고에 필요한 커뮤니티 반응을 보수적으로 추출하는 제품을 만들기 위한 경계입니다.

블로그는 검색 결과 전체를 수집하기보다 사전에 선정한 블로그 source registry를 추적합니다. 블로그/유튜브/뉴스/공시는 커뮤니티 반응의 원문이 아니라 맥락 확인용 외부 이벤트로 다루며, 제목, 출처, 시간, 링크, 관련 키워드 중심으로 표시합니다.

## 도메인/layer 접점

- `community`: 소스 구조를 구분하고, 안전한 방식으로 제한 원문과 메타데이터를 수집하며 반응 방향 후보를 관리합니다.
- `stock`: 종목 alias, symbol, market code 정규화 기준을 제공합니다.
- `indicator`: 반응 지표, 30분 집계, 개미 심리 지수, 통합 지표 기반 성과 비교용 원천 지표를 만듭니다.
- `market`: 반응 지표와 함께 볼 시세, 거래량, 수급 데이터를 제공합니다.
- `agent`: 반응 지표와 시세를 읽어 paper trading 판단과 성과 비교를 만듭니다.
- `layers/ui`: 커뮤니티 반응을 투자 참고 화면에서 이해하기 쉽게 보여줍니다.
- `layers/ops`: 용어, 문서, Notion, PR 기록이 같은 기준을 따르는지 점검합니다.
