# 커뮤니티 반응 용어 가이드

이 문서는 너나사 부동산에서 커뮤니티 글을 분석할 때 쓰는 제품 용어와 수집 전략을 정리합니다.

## 용어 기준

너나사 부동산은 `감성`이라는 표현을 대표 용어로 쓰지 않습니다. 서비스가 보여주려는 것은 막연한 기분 분석이 아니라, 지역과 단지에 대해 관찰되는 기대, 우려, 쟁점을 데이터로 정리한 결과입니다.

대표 용어:

- `커뮤니티 반응`: 사용자 화면에서 쓰는 대표 표현
- `반응 지표`: 화면과 API 설명에서 쓰는 집계 지표
- `반응 방향`: 단일 게시글 안에서 특정 지역/단지가 어떤 방향으로 언급되는지 나타내는 값
- `지역/단지 반응 지표`: 여러 커뮤니티 반응을 target/window 단위로 묶어 압축한 관찰 지표
- `종합 반응`: target/window 단위로 사용자에게 먼저 보여주는 하나의 관찰 상태
- `주요 쟁점`: 종합 반응을 만든 이유나 말거리의 비율

사용자 화면에서는 `커뮤니티 반응`과 `반응 지표`를 우선합니다. `신호`는 매수/매도 판단처럼 오해될 수 있으므로 대표 화면 문구로 쓰지 않습니다.

반응 방향 표시값:

| 내부 값 | 사용자 표시 | 의미 |
| --- | --- | --- |
| `expectation` | 기대 | 교통, 개발, 공급 부족, 학군, 정책 변화 등에 대한 기대 반응 |
| `concern` | 우려 | 전세, 대출, 공급, 정책 불확실성, 가격 부담 등에 대한 우려 반응 |
| `neutral` | 중립 | 명확한 방향이 없거나 정보 공유에 가까운 반응 |

향후 코드/DB 이름을 정리할 때는 `reactionDirection`을 우선 후보로 둡니다.

1차 룰 기반 쟁점 분류:

- `transport`: GTX, 지하철, 역세권, 교통, 개통, 노선, 철도
- `jeonse`: 전세, 역전세, 전세난, 보증금
- `loan_rate`: 대출, 금리, DSR, 이자, 상환
- `supply`: 미분양, 공급과잉, 입주폭탄, 물량 부담
- `policy`: 규제, 정책, 토허제, 세금
- `school`: 학군, 초품아, 학교
- `redevelopment`: 재건축, 재개발, 정비구역

이 분류는 AI 최종 판단이 아니라 `realestate-reaction-observations`의 baseline입니다. OpenAI provider가 붙더라도 출력은 `reactionDirection`과 `issues` shape를 유지합니다.

## 피해야 할 표현

- `추천`, `사라`, `팔아라`
- `수익 보장`
- `진입`
- `시그널 확정`
- `오를 지역`
- `무조건 오른다`
- `청약 넣어라`
- `대출 받아라`
- `지금 사야 한다`

허용되는 표현의 기준은 단어 자체가 아니라 말하는 주체와 기능입니다. 예를 들어 `대출 규제 언급 증가`, `청약 관련 글 증가`는 관찰 데이터입니다. 반대로 서비스가 특정 행동을 평가하거나 유도하면 제품 결론으로 읽힐 수 있어 피합니다.

짧고 강한 카피를 쓰더라도 대상은 사용자가 아니라 데이터 상태와 관찰된 괴리여야 합니다.

예시:

- `언급은 뜨거운데 실거래는 아직 느립니다`
- `기대는 교통에 몰렸고, 우려는 전세에 몰렸습니다`
- `표본이 작아 방향 판단은 보류합니다`
- `뉴스는 늘었지만 실제 시장 사실은 확인이 더 필요합니다`

## 소스 구조별 수집 전략

모든 커뮤니티를 같은 방식으로 수집하지 않습니다. 소스가 시장 전체 이야기를 섞어 보여주는지, 지역/단지별 공간으로 나뉘는지에 따라 역할을 나눕니다.

### 소스 활성화 상태

| 상태 | 수집 실행 | 공개 화면 | 기준 |
| --- | --- | --- | --- |
| `enabled` | 허용 | 집계 지표 표시 가능 | 운영 정책 검토가 끝난 소스 |
| `public-demo-only` | 공개 환경에서는 외부 요청 금지 | fixture/sample 표시 가능 | 공개 시연용 대체 데이터 |
| `local-research-only` | 로컬/비공개 연구에서만 허용 | 공개 노출 금지 | 검토 전 실험 소스 |
| `disabled` | 금지 | 숨김 또는 비활성 표시 | 위험, 차단, 접근 불가, 미검토 소스 |

새 소스는 기본적으로 `disabled`에서 시작합니다. 공개 배포 전에는 별도 검토 없이 `enabled`로 올리지 않습니다.

### 일반 게시판형

역할:

- 시장 전체 반응 흐름 파악
- 새로 언급이 튀는 지역/단지 발견
- 정책, 교통, 전세, 공급, 청약 같은 쟁점 후보 생성

수집 방식:

- 최근 글 목록을 주기적으로 수집합니다.
- 제목과 제한 snippet에서 지역명, 단지명, 별칭을 인식합니다.
- 언급량 증가율, 키워드 변화, 반응 방향 분포를 이용해 후보 target을 만듭니다.
- 인기글, 댓글 많은 글, 조회수 상위글은 일반 최신 글 수집을 대체하지 않고 확산 확인 레이어로 수집합니다.
- 조회수, 추천수, 댓글수는 source마다 의미가 다르므로 raw 수치끼리 직접 비교하지 않습니다.

### 대상 게시판형

대상 예시:

- 지역별 부동산 게시판
- 단지별 게시판 또는 커뮤니티
- 뉴스/컬럼 댓글 영역

역할:

- 특정 지역/단지에 대한 깊은 반응 확인
- 일반 게시판형에서 발견한 후보 target 검증
- 지역별 기대, 우려, 이슈 반응 보강

수집 방식:

- 모든 지역/단지를 전수 순회하지 않습니다.
- `CrawlTarget` 큐로 우선순위를 관리합니다.
- 우선순위는 일반 게시판 언급 증가, 사용자 관심 지역/단지, 정책/개발 이벤트, 최근 활동량, 실패 backoff를 함께 반영합니다.

## 반응 데이터 사용 방식

| 목적 | 우선 소스 | 보조 소스 |
| --- | --- | --- |
| 시장 전체 반응 | 일반 게시판형 | 뉴스/컬럼, 정책 이벤트 |
| 관심 급등 target 후보 | 일반 게시판형 | 대상 게시판형, market fact |
| 지역/단지 반응 상세 | 대상 게시판형 + 일반 게시판형 | AI 재서술 근거 |
| 이슈 확산 확인 | 인기글/댓글/조회수 상위글 | 뉴스/컬럼 링크 |
| 유사 과거 비교 | 반응 지표, issueMix | 이후 market fact 흐름 |

블로그/뉴스/컬럼은 커뮤니티 반응의 원문이 아니라 맥락 확인용 외부 이벤트로 다루며, 제목, 출처, 시간, 링크, 관련 키워드 중심으로 표시합니다.

## 도메인/layer 접점

- `community`: 소스 구조를 구분하고, 안전한 방식으로 제한 원문과 메타데이터를 수집하며 반응 방향 후보를 관리합니다.
- `realestate`: 지역/단지 alias, 법정동 코드, 단지 id 정규화 기준을 제공합니다.
- `indicator`: 반응 지표, window snapshot, 표본 신뢰도, 쟁점 비율을 만듭니다.
- `agent`: 반응 지표와 market fact를 읽어 평가와 근거 로그를 만듭니다.
- `layers/ui`: 커뮤니티 반응을 부동산 관찰 화면에서 이해하기 쉽게 보여줍니다.
- `layers/ops`: 용어, 문서, Notion, PR 기록이 같은 기준을 따르는지 점검합니다.
