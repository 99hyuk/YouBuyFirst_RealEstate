# 종목 상세 팩트폭격 카피 가이드

이 문서는 특정 종목 상세 화면 상단에 노출되는 짧고 강한 헤드라인의 기준을 정리합니다. 이 카피는 커뮤니티 반응 요약이 아니라, 실제 시황과 종목 상태를 분석해 만드는 `종목 상태 한줄평`입니다.

목표는 사용자가 종목 상세에 들어오자마자 "지금 이 종목이 어떤 상태인지"를 웃기지만 날카롭게 이해하게 만드는 것입니다. 단순 투자 자문이 아니라 가격, 기술 지표, 재무, 성장성, 뉴스/이벤트, 애널리스트 컨센서스 같은 관찰 가능한 데이터의 압축 표현입니다.

## 생성 역할

- 종목 상세 첫 화면의 상단 헤드라인 문구를 생성합니다.
- 차트, CAN SLIM, 기술 지표, 재무 지표, 뉴스/공시, 애널리스트 의견을 보기 전의 요약 훅입니다.
- 사용자가 클릭한 종목의 상태를 빠르게 판단하게 돕되, `사라/팔아라` 같은 행동 지시로 쓰지 않습니다.
- 커뮤니티 반응 문구와 섞지 않습니다. 커뮤니티 반응은 별도 섹션에서 `커뮤니티는 이렇게 보고 있음`으로 표시합니다.

화면 표현 기준과 레퍼런스 이미지는 `docs/layers/ui/STOCK_DETAIL_BANNER.md`가 소유합니다.

## 입력 데이터

헤드라인은 최소 2개 이상의 근거 묶음이 같은 방향을 가리킬 때만 강한 톤을 씁니다.

| 묶음 | 예시 |
| --- | --- |
| 가격/추세 | 3개월/12개월 수익률, 고점 대비 낙폭, EMA20/50/200 위치, VWAP 괴리 |
| 거래/수급 | 거래량 급증, 유동성, 변동성, 매수/매도 잔량, 기관/외국인 순매수 |
| 기술 지표 | RSI, ADX, ATR, RS rating, 지지/저항 근접도 |
| 재무/성장 | 매출 성장, EPS 성장, ROE, 마진, 부채비율, 현금흐름 |
| 밸류에이션 | PER, PBR, PSR, 동종 업계 대비 과열/저평가 |
| 이벤트 | 실적 발표, 가이던스, 공시, 규제, 제품/계약 뉴스 |
| 컨센서스 | 애널리스트 목표가, 투자의견 변화, 실적 추정치 변화 |

강한 카피 아래에는 근거를 작은 보조 문장으로 붙입니다.

예시:

- `실적서프라이즈 -188.5%, 내부자 매도, 공매도 18.1%, 기관보유 46%, 컨센서스 보류`
- `12개월 수익률 -37.8%, RS 등급 54, VWAP 위 +10.2%, EPS 성장률 없음`

## 톤 원칙

이 화면의 톤은 `짠맛`, `팩트폭격`, `야유 섞인 데이터 브리핑`입니다. 다만 농담의 대상은 사용자가 아니라 종목의 현재 상태와 지표의 모순이어야 합니다.

좋은 방향:

- 지표가 서로 어긋나는 부분을 찌릅니다.
- 기대감과 실제 성과의 간극을 웃기게 표현합니다.
- 차트, 재무, 수급, 뉴스의 약점을 근거와 함께 압축합니다.
- 한 문장은 짧고 세게, 보조 문장은 데이터로 차갑게 씁니다.

피해야 할 방향:

- 기본/개인화 화면에서 `너`, `네 안목`, `사라`, `팔아라`처럼 사용자의 판단이나 행동을 직접 공격합니다.
- 근거가 하나뿐인데 확정적인 조롱을 합니다.
- 수익 보장, 손실 확정, 상장폐지 확정처럼 미래를 단정합니다.
- 뉴스 제목이나 커뮤니티 원문을 서비스 결론처럼 재사용합니다.

사용자 제공 레퍼런스처럼 `안목`, `매수`, `물림`을 건드리는 강한 문구는 `roast` 단계로 분리합니다. 공개 데모 또는 검색으로 들어온 종목 상세에서는 쓸 수 있지만, 사용자가 실제로 보유 중인 종목이나 개인화 알림에서는 기본값으로 쓰지 않습니다.

## 강도 단계

| 단계 | 용도 | 예시 |
| --- | --- | --- |
| `normal` | 일반 공개 기본값 | `지표가 아직 회복을 허락하지 않음` |
| `spicy` | 데이터가 명확히 나쁠 때 | `기대감은 상한가, 숫자는 아직 장전` |
| `roast` | 종목 상태가 여러 지표에서 동시에 나쁠 때 | `분할매수 한다고 했는데 물타기만 남은 차트` |

`roast` 단계는 재미는 강하지만 사용자 불쾌감도 커질 수 있으므로, 보유 종목/관심종목 개인화 화면에서는 기본값으로 쓰지 않습니다. 검색으로 들어간 공개 종목 상세나 데모 화면에서 우선 사용합니다.

## 카피 예시

사용자 제공 레퍼런스 원문:

| 레퍼런스 | 헤드라인 | 운영 메모 |
| --- | --- | --- |
| Palantir | `종목 고르다 이거 골랐으면 안목을 의심해봐` | 가장 공격적인 `roast`. 데모/공개 상세용으로만 우선 사용 |
| NuScale | `분할매수 한다고 했는데 전부 다 물린 거 맞지?` | 물림/평단 이슈를 찌르는 `roast`. 보유 종목 개인화에서는 순화 |
| AST SpaceMobile | `이거 매수하는 건 돈에 대한 모독임` | 매우 강한 `roast`. 근거가 충분한 데모 문구로 보관하고 운영 기본값은 순화 |

위 문구들은 제품이 지향하는 `짠맛`의 원형입니다. 다만 실제 운영 문구로 바로 쓰기보다, `headlineTone: "roast"`와 근거 지표가 함께 있을 때만 사용합니다.

나쁜 상태:

- `분할매수 한다고 했는데 물타기만 남은 차트`
- `종목은 미래를 말하는데, 숫자는 아직 과거에 있음`
- `기대감은 상한가, 실적표는 하한가 근처`
- `차트는 바닥을 찾는 중인데 재무도 같이 누워 있음`
- `성장주라고 부르기엔 성장률이 먼저 퇴근함`

과열 상태:

- `가격은 달렸고, 근거는 아직 신발 끈 묶는 중`
- `차트는 뜨거운데 이익은 아직 미지근함`
- `스토리는 화려한데 밸류에이션은 이미 VIP석`
- `기대감은 선반영, 실적은 후불 결제`

애매한 상태:

- `좋다고 하긴 애매하고, 나쁘다고 하기엔 끈질김`
- `지표끼리 서로 다른 회의실에 앉아 있음`
- `반등 냄새는 나는데 확인 도장은 아직 안 찍힘`
- `기술 지표는 웃고, 재무 지표는 정색 중`

좋은 상태:

- `숫자가 먼저 말하고, 차트가 따라오는 중`
- `조용한데 지표는 꽤 성실함`
- `실적이 받쳐주니 차트도 덜 민망함`
- `과열은 아닌데 체력은 살아 있음`

## 생성 규칙

1. 먼저 `stockHealthScore`, 추세, 재무, 수급, 뉴스 리스크를 계산합니다.
2. 가장 강한 약점 또는 강점을 1개 고릅니다.
3. 반대 근거가 강하면 단정하지 않고 `애매함`, `엇갈림`, `확인 필요` 톤으로 낮춥니다.
4. 헤드라인은 30자 안팎, 보조 문장은 근거 3-5개로 제한합니다.
5. 보조 문장에는 기준일과 기간을 붙입니다.
6. 사용자가 보유 중인 종목에는 개인을 조롱하는 표현을 쓰지 않습니다.
7. 공개 배포 전에는 `roast` 단계 카피를 별도 fixture로 모아 리뷰합니다.
8. 레퍼런스 이미지와 같은 검은 배너는 `layers/ui`가 담당하고, 근거 필드는 `market`, `indicator`, `agent` 도메인과 계약합니다.

## 백엔드/API 계약 후보

프론트는 검은 배너 UI를 담당하지만, 배너 문구와 근거는 backend/API 계약에 들어가야 합니다. 처음에는 stock detail 응답 안에 포함하고, 이후 비용이나 캐시 정책이 필요하면 별도 endpoint로 분리합니다.

후보 endpoint:

- `GET /api/stocks/{symbol}/detail`
- 또는 `GET /api/stocks/{symbol}/roast-headline`

트랙별 책임:

| 트랙 | 책임 |
| --- | --- |
| `backend` | API 응답 DTO, 캐시, `asOf`, stale 상태, Swagger/OpenAPI 문서 |
| `market` | 가격, 수익률, EMA/VWAP, 거래량, 호가/수급성 근거 |
| `indicator` | 기술/재무/밸류에이션 지표, 지표 방향성, stock health score 후보 |
| `agent` | 근거 묶음을 읽고 headline tone과 문구 후보를 선택하거나 생성 |
| `layers/ui` | 검은 배너 UI, 모바일 줄바꿈, tone별 색/강도, 근거 노출 |

최소 응답 필드:

| 필드 | 의미 |
| --- | --- |
| `headlineTone` | `normal`, `spicy`, `roast` |
| `headline` | 상단 큰 한줄평 |
| `subtitle` | 근거 지표 3-5개를 붙인 작은 보조 문장 |
| `evidence` | 문구를 만든 근거 배열 |
| `asOf` | 판단 기준 시각 |
| `dataQuality` | stale, partial, mock, insufficient 같은 상태 |
| `personalizedSafe` | 보유 종목/개인화 화면에 그대로 노출 가능한지 |
| `disclaimer` | 투자 자문이 아니라 공개 데이터 기반 상태 요약이라는 고지 |

`roast` 톤은 `evidence`가 2개 이상이고, 서로 다른 묶음이 같은 방향을 가리킬 때만 응답합니다. 근거가 부족하면 `normal` 또는 `spicy`로 낮춥니다.

```json
{
  "symbol": "SMR",
  "asOf": "2026-05-18T09:30:00+09:00",
  "headlineTone": "roast",
  "headline": "분할매수 한다고 했는데 물타기만 남은 차트",
  "subtitle": "12개월 수익률 -37.8%, RS 등급 54, 내부자 매도 증가, 컨센서스 보류",
  "dataQuality": "complete",
  "personalizedSafe": false,
  "evidence": [
    {
      "type": "technical",
      "label": "12개월 수익률",
      "value": "-37.8%",
      "direction": "bad"
    },
    {
      "type": "fundamental",
      "label": "EPS 성장률",
      "value": "없음",
      "direction": "bad"
    }
  ],
  "disclaimer": "투자 자문이 아니라 공개 데이터 기반 종목 상태 요약입니다."
}
```

이 데이터는 `front`가 화면에 보여주고, `market`, `data`, `agent` 트랙이 어떤 필드를 실제로 제공할지 계약을 나눠 확정합니다.
