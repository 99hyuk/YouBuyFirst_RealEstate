# simulation 작업 지침

> Legacy stock reference: 이 부동산 전용 프로젝트에서는 새 simulation 작업을 열지 않습니다. 기존 구조 참고나 후속 삭제 후보 정리에만 사용합니다.

simulation은 가상 계좌, 예수금, 주문/체결, 원장, 포트폴리오, 수익률 계산을 소유합니다.

## 시작

- 계좌/주문/체결/원장 contract가 걸리면 `README.md`의 관련 섹션만 봅니다.
- 판단 주체나 전략 입력이 문제면 `agent`를 봅니다.
- 시세 가격 source가 문제면 `market`을 봅니다.

## 경계

- 실거래, 실제 결제, 실제 투자 권유 기능을 만들지 않습니다.
- simulation은 장부의 정합성과 재현 가능한 체결 기록을 담당하고, 투자 판단 자체는 `agent`가 담당합니다.
- 화면의 포트폴리오 배치와 empty/error 상태는 `layers/ui`가 소유합니다.

## 기록

- 돈처럼 보이는 값은 원장 기반으로 추적하고, 단순 balance overwrite를 기본값으로 두지 않습니다.
- 주문/체결 상태값, idempotency key, 실패 사유, 정산 기준 시간을 함께 남깁니다.
- 취업용 기술 경험 후보가 될 정합성/트랜잭션/동시성 판단은 Notion 개발자 기술 경험 DB나 관련 문서로 승격합니다.
